### 📮 우체국을 통해 편지를 주고받기

텔레그램 봇은 **우체국을 통해 편지를 주고받는 것**과 아주 비슷하다냥.

사용자가 보낸 편지(메시지)는 텔레그램이라는 **우체국(서버)**에 먼저 도착한다냥. 내 프로그램은 그 우체국에 가서 편지를 받아 읽은 뒤, 답장을 써서 다시 우체국에 맡긴다냥. 그러면 우체국이 사용자에게 답장을 대신 전달해주는 거다냥. 우리는 사용자에게 직접 가지 않아도, 우체국만 잘 이용하면 대화를 주고받을 수 있다냥!

<Callout type="analogy">
  **비유로 이해하기 — 우체국 메시지 배달**

  | 우체국 이야기 | 텔레그램 봇 |
  |---|---|
  | 사용자가 쓴 편지 | 사용자가 보낸 메시지 |
  | 편지를 모아두는 우체국 | 텔레그램 서버 |
  | 편지를 받아 읽고 답장 쓰는 나 | 내 봇 코드(파이썬 프로그램) |
  | 답장을 사용자에게 배달 | 텔레그램이 답장 전송 |

  내 코드는 우체국(텔레그램 서버)을 통해서만 사용자와 대화한다냥!
</Callout>

### 🔑 봇 토큰 = 내 우체통을 여는 열쇠

봇을 만들려면 먼저 텔레그램의 **BotFather**(봇들의 아빠)에게 가서 봇을 하나 만들어달라고 부탁한다냥. 그러면 BotFather가 **토큰(token)**이라는 긴 비밀번호를 하나 발급해준다냥.

이 토큰은 "이 우체통은 내 것이에요!"라고 증명하는 **열쇠**다냥. 이 열쇠가 있어야 내 코드가 우체국에서 내 봇의 편지를 받아올 수 있다냥.

<Callout type="important">
  ⚠️ **봇 토큰은 절대 남에게 보여주면 안 된다냥!**

  토큰은 **API 키와 똑같은 보안 정보**다냥. 남이 이 열쇠를 알면 내 봇을 마음대로 조종할 수 있다냥. 그래서 코드를 친구와 공유하거나 인터넷에 올릴 때는 **토큰이 노출되지 않도록 꼭 조심**해야 한다냥!
</Callout>

## 3단계 · 메아리 봇으로 기본 흐름 익히기 (3분)

### 🦜 메아리부터 배우는 외국어

외국어를 처음 배울 때를 떠올려보자냥. 보통 상대가 한 말을 **그대로 따라 하는 연습**부터 시작하지 않냥? "Hello"라고 하면 "Hello"라고 따라 하는 식이라냥.

봇도 똑같다냥! 일단 사용자가 보낸 말을 그대로 따라 답장하는 **메아리 봇**부터 만들어서, "메시지를 받고 → 답장을 보낸다"는 **기본 흐름**을 확실하게 몸에 익히는 거다냥. 이 흐름만 이해하면, 나중에 **답장 부분만 AI로 바꾸면** 똑똑한 봇이 뚝딱 완성된다냥! 🎉

### 🐣 메아리 봇 기본 코드

```python
from telegram import Update
from telegram.ext import Application, MessageHandler, CommandHandler, filters, ContextTypes

TELEGRAM_TOKEN = "여기에 BotFather에서 받은 토큰 입력"

# /start 명령어에 응답하는 함수
async def start(update: Update, context: ContextTypes.DEFAULT_TYPE):
    await update.message.reply_text("안녕하세요! 메시지를 보내면 따라 말해드릴게요.")

# 일반 메시지에 응답하는 함수
async def echo(update: Update, context: ContextTypes.DEFAULT_TYPE):
    user_text = update.message.text
    await update.message.reply_text(user_text)

# 봇 실행
app = Application.builder().token(TELEGRAM_TOKEN).build()
app.add_handler(CommandHandler("start", start))
app.add_handler(MessageHandler(filters.TEXT & ~filters.COMMAND, echo))

print("봇이 실행 중입니다...")
app.run_polling()
```

<Callout type="tip">
  💡 **코드 흐름을 한 줄로 정리하면**

  - `start` 함수: 사용자가 `/start`를 누르면 인사말을 답장한다냥.
  - `echo` 함수: 사용자가 보낸 글자(`update.message.text`)를 그대로 답장한다냥.
  - `add_handler`: "이런 메시지가 오면 이 함수를 써라"라고 봇에게 알려주는 부분이라냥.
  - `run_polling()`: 봇을 **계속 켜둔 상태**로 만들어 메시지를 기다리게 한다냥!
</Callout>

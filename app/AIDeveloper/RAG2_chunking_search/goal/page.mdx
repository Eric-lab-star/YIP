## 1단계 · 잠깐, 책 한 권을 통째로 임베딩해도 되냥? (5분)

<NyangSpeech>
  지난 시간에는 짧은 **문장 하나**를 벡터로 바꿔봤지? 그런데 여기서 질문이 하나 생긴다냥.
  "AI에게 참고시킬 자료가 **책 한 권처럼 아주 길다면**, 그 전체를 한 번에
  임베딩해도 괜찮을까냥?" 🤔 같이 생각해보자냥.
</NyangSpeech>

두꺼운 책 한 권을 통째로 "이 책이 무슨 내용이야?"라고 **단어 하나로** 표현하라고 하면 어떻게 될까냥? 너무 뭉뚱그려져서 "음… 그냥 책?" 정도밖에 안 나온다냥. 책 안에는 요리 이야기, 여행 이야기, 과학 이야기가 다 섞여 있는데, 그걸 좌표 하나로 찍으면 **핵심 의미가 흐려져 버린다냥.**

<Callout type="important">
  긴 글을 통째로 임베딩하면 두 가지 문제가 생긴다냥.

  - 너무 긴 글은 임베딩할 때 **핵심 의미가 희석**된다냥. (여러 주제가 한 좌표에 뭉쳐버림)
  - AI가 한 번에 처리할 수 있는 **글자 수에도 한계**가 있다냥.

  그래서 긴 문서는 **잘게 나눠서** 다뤄야 한다냥!
</Callout>

## 2단계 · 청킹(Chunking)이 뭐냥? (5~7분)

<NyangSpeech>
  그래서 등장하는 게 바로 **청킹(Chunking)**이다냥! 어려운 말 같지만
  뜻은 아주 간단하다냥. 긴 문서를 **작은 조각(청크)으로 잘게 나누는 작업**이라냥. 🐟
</NyangSpeech>

**청킹**이란 긴 문서를 **의미 단위**(문단, 일정 글자 수 등)로 잘라서 여러 개의 작은 조각으로 나누는 작업이다냥. 잘라진 조각 하나하나를 **청크(chunk)**라고 부른다냥.

### 📚 책 한 권을 챕터별로 나누어 책장에 꽂기

<Callout type="analogy">
  **비유로 이해하기**

  두꺼운 책 한 권을 통째로 "이 책 무슨 내용이야?"라고 하나의 단어로 표현하면 너무 뭉뚱그려져서 정확하지 않다냥.

  그래서 우리는 책을 **챕터(또는 문단) 단위로 나눠서** 각각 책장에 따로 정리한다냥.

  - **청킹** = 책을 **챕터별로 나누는 작업** 📖
  - 그리고 지난 시간 배운 **임베딩**으로 각 챕터(청크)를 **책장의 좌표에 꽂아두기** 📍
  - 질문이 들어오면 → "이 질문과 **가장 가까운 챕터**가 뭐지?"를 빠르게 찾을 수 있게 된다냥!
</Callout>

<Callout type="tip">
  💡 **청크 크기는 "적당히"가 중요하다냥!**

  - 청크가 **너무 작으면** → 앞뒤 맥락이 부족해진다냥.
  - 청크가 **너무 크면** → 다시 의미가 희석된다냥.

  그래서 "적절한 크기"를 찾는 게 포인트라냥. 오늘은 일단 글자 수 기준으로 가볍게 나눠볼 거다냥!
</Callout>

## 3단계 · 오늘 만들 것 — "검색기"가 뭐냥? (3분)

<NyangSpeech>
  자, 오늘 우리가 직접 만들어볼 건 바로 **"검색기"**다냥. 긴 텍스트를 청크로 나누고,
  각 청크를 임베딩한 다음, 질문과 **가장 유사한 청크**를 찾아내는 도구라냥! 🐾
</NyangSpeech>

<StepCard step="오늘의 과제" title="질문에 가장 잘 맞는 청크를 찾아주는 검색기 만들기">

긴 텍스트 문서를 **여러 청크로 나누고**, 각 청크를 **임베딩**한 뒤, 질문과 **가장 유사한 청크**를 찾아내는 검색기를 만들어본다냥.

검색기가 동작하는 순서는 이렇다냥.

1. 문서를 청크로 **나누기** (청킹) ✂️
2. 각 청크를 **임베딩**으로 변환하기 📍
3. 질문도 **임베딩**으로 변환하기 ❓
4. 질문과 각 청크의 **유사도 비교**하기 📏
5. 가장 유사한 청크를 **결과로 돌려주기** 🎁

</StepCard>

### 🐣 청크로 나누는 기본 코드 미리보기

긴 문서를 일정 글자 수만큼 잘라서 청크 리스트로 만드는 함수는 이런 모양이다냥. (자세한 빈칸 채우기는 실습 페이지에서 해본다냥!)

```python
def split_into_chunks(text, chunk_size=200):
    chunks = []
    for i in range(0, len(text), chunk_size):
        chunk = text[i:i + chunk_size]
        chunks.append(chunk)
    return chunks
```

`chunk_size`만큼씩 텍스트를 잘라서 차곡차곡 `chunks` 리스트에 담는 거라냥. 200으로 정하면 200글자씩, 100으로 정하면 100글자씩 잘린다냥!

<Callout type="note">
  📝 오늘은 **"검색"까지만** 진행한다냥. AI가 찾아낸 자료를 보고 **답변을 자연스럽게 만들어주는 것**은 다음 차시에서 배울 거다냥. 오늘은 "정확한 자료를 찾아주는" 데까지 집중하자냥!
</Callout>

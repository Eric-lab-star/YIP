오늘은 그 마음을 완전히 들어주는 파일, **`.ipynb`(주피터 노트북) 파일**을 VS Code에서 직접 만들어보고, 너희가 알고 있던 `.py` 파일이랑 뭐가 다른지 비교해볼 거다냥. 어려워 보여도 걱정 마냥~ 내가 다 쉬운 비유로 설명해줄 거니까!

---

## 1. `.py` 파일은 "한 번에 다 먹는 츄르" 같다냥

`.py` 파일에 코드를 쓰면, 보통 **위에서부터 아래까지 전체를 한 번에 실행**한다냥.

마치 내가 츄르 한 봉지를 츄르륵~ 한 번에 다 빨아먹는 것처럼, 코드도 처음부터 끝까지 다 실행되고 나서야 **결과를 한꺼번에** 볼 수 있다냥. 중간에 "음, 이 부분은 어떻게 됐지?" 궁금하면, 따로 멈춰서 확인해야 한다냥.

```python
# my_recipe.py
재료_준비()
재료_썰기()
볶기()
간_맞추기()
완성()
```

→ 실행하면 끝까지 쭉~ 다 돌아간 다음에야 결과가 보인다냥.

---

## 2. `.ipynb` 파일은 "한 알씩 음미하는 간식 시간" 같다냥

반면에 `.ipynb` 파일은 코드를 **작은 칸(이걸 "셀(cell)"이라고 부른다냥)** 으로 나눠서 쓴다냥.

내가 간식을 한 알씩 천천히 먹으면서 "음~ 이건 맛있다냥" "이건 좀 짜다냥" 하고 매번 반응하는 것처럼, `.ipynb`도 **한 칸(셀)을 실행할 때마다 그 칸 바로 아래에 결과가 쏙!** 나타난다냥.

- 한 칸에 코드를 쓰고 실행하면 → **그 자리에서 바로 결과 확인 가능**이다냥
- 코드 칸 사이사이에 **설명 글(마크다운 셀)** 을 자유롭게 끼워 넣을 수 있다냥
- 어떤 칸은 다시 실행하고, 어떤 칸은 그냥 건너뛰는 것도 가능하다냥

그래서 `.ipynb`는 "코드 + 결과 + 설명 글"이 마치 **나의 간식 일기장**처럼 차곡차곡 쌓이는 느낌이다냥! 데이터를 살펴보거나 AI 실습을 할 때, 한 단계씩 확인하면서 진행하기에 딱이라냥.

---

## 3. 한눈에 비교해보자냥

|               | `.py` 파일 (일반 코드 파일) | `.ipynb` 파일 (노트북 파일)       |
| ---           | ---                         | ---                               |
| 비유 | 츄르 한 봉지 한입에 | 간식 한 알씩 음미하기 |
| 실행 방식     | 전체를 한 번에 실행         | 칸(셀) 단위로 따로 실행           |
| 결과 확인     | 다 끝나야 결과 확인         | 칸마다 바로 결과 확인             |
| 설명 글 추가  | 주석(`#`)으로만 가능        | 마크다운 셀로 자유롭게 작성       |
| 어울리는 작업 | 완성된 프로그램 만들기      | 실습, 데이터 탐색, 학습 과정 기록 |

## 4. VS Code에서 `.ipynb` 파일 직접 만들어보자냥!

자, 이제 발톱을 꺼내고(?) 직접 만들어볼 시간이다냥! 차근차근 따라오면 5분도 안 걸린다냥~

1. **VS Code를 열어보자냥.**
2. 왼쪽 위 메뉴에서 **`File(파일)` → `New File...(새 파일)`** 을 클릭하자냥.
3. 파일 형식을 고르는 화면에서 **"Jupyter Notebook"** 을 선택하자냥. (만약 안 보이면, 파일 이름을 `practice.ipynb` 로 직접 입력해서 만들어도 괜찮다냥.)
4. 새 노트북이 열리면, 빈 칸(셀)이 하나 보일 거다냥. 여기에 간단한 코드를 써보자냥.

```python
print("야옹, 나는 .ipynb 파일이다냥!")
```

5. 코드를 쓴 칸 옆에 있는 **▶ (재생 버튼)** 을 누르거나, **`Shift + Enter`** 키를 눌러보자냥.
6. 칸 바로 아래에 결과(`야옹, 나는.ipynb 파일이다냥!`)가 쏙 나타나는 걸 확인해보자냥!
7. 화면 위쪽의 **`+ Markdown`** 버튼을 눌러서 설명 글을 적는 칸도 한번 만들어보자냥. 예를 들어 "이 셀은 인사 메시지를 출력하는 코드다냥" 같은 메모를 남겨보는 거다냥.

> 만약 실행했을 때 "커널을 선택하세요(Select Kernel)"라는 안내창이 뜨면, Python을 선택하면 된다냥. 처음 한 번만 설정하면 그 다음부터는 자동으로 연결된다냥.

## 5. 직접 해보는 미니 실습이다냥!

아래 순서대로 따라 하면서, `.py`랑 `.ipynb`의 차이를 직접 느껴보자냥.

3. 이번엔 2번 셀만 다시 실행해보자냥. 무슨 일이 일어나냥?
4. 같은 코드를 `.py` 파일에 한꺼번에 써서 실행해보고, `.ipynb`와 비교해보자냥. 어떤 점이 더 편했냥?

## 6. 정리하고 가자냥!

- `.py` 파일은 **"처음부터 끝까지 한 번에 실행하는 완성된 코드"** 에 딱이다냥.
- `.ipynb` 파일은 **"한 단계씩 실행하고, 결과를 바로 확인하고, 설명 글까지 남길 수 있는 노트"** 다냥.
- 그래서 AI나 데이터를 다루는 실습에서는 `.ipynb`가 정말 자주 쓰인다냥. 한 단계씩 결과를 보면서 "왜 이렇게 됐을까냥?" 하고 생각해볼 수 있기 때문이다냥.

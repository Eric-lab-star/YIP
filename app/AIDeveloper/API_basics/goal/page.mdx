### 🍔 키오스크로 이해하는 API

햄버거 가게에 가면 **키오스크**가 있지? 우리가 화면을 터치해서 주문하면, 키오스크가 주방에 "빅맥 하나 주세요!" 하고 전달해준다냥.

우리가 주방 안으로 직접 들어가서 요리사한테 말할 필요가 없는 거다냥. 키오스크가 **중간에서 연결해주는 역할**을 하는 거다냥!

> 🐱 API도 똑같다냥. 내가 만든 프로그램과 다른 서비스(예: AI, 날씨 데이터, 지도 등) 사이에서 **"주문을 전달해주는 키오스크"** 역할을 하는 거다냥!

<Callout type="analogy">
  **비유로 이해하기**

  | 키오스크 주문 | API |
  |---|---|
  | 손님 (나) | 내가 만든 프로그램 (코드) |
  | 키오스크 | API |
  | 주방 (요리사) | 제미나이, 날씨 서버 등 외부 서비스 |
  | 주문한 음식 | API가 가져다주는 데이터 (AI 답변, 날씨 정보 등) |

  손님이 직접 주방에 들어가지 않아도 되듯이, 내 코드가 제미나이 서버 안으로 들어가지 않아도 된다냥!
</Callout>

### 📱 배달 앱으로 한 번 더 이해하기

배달 앱을 쓸 때, 앱이 지도 API를 써서 식당 위치를 보여주고, 결제 API를 써서 카드 결제를 처리하고, 날씨 API를 써서 "오늘 비가 오니까 우산 챙겨요"를 보여준다냥.

이처럼 하나의 앱 안에 **여러 개의 API**가 동시에 쓰이는 경우가 많다냥!

<Callout type="tip">
  💡 **핵심 한 줄 요약**

  API = 내 프로그램이 다른 서비스에 "요청"을 보내고 "결과"를 받아오는 창구다냥!
</Callout>

### 📌 실생활 비교 예시

<StepCard step="상황" title="오늘 서울 날씨를 AI한테 물어보고 싶다냥!">

**제미나이 앱을 쓰면?**

```python print("hello")
나 → 제미나이 앱 화면 → 직접 타이핑: "오늘 서울 날씨 알려줘"
                       → 제미나이가 답변
                       → 내가 눈으로 읽고 끝
```

**제미나이 API를 쓰면?**

```python
날씨 앱(내 코드) → 날씨 API에서 기온 데이터 가져옴
               → 제미나이 API에 자동으로 전송: "서울 기온이 32도야. 오늘 날씨 코멘트 써줘"
               → 제미나이가 답변을 내 코드에 돌려줌
               → 앱 화면에 예쁘게 표시!
```

코드가 알아서 척척 한다냥! 사람이 일일이 타이핑할 필요가 없다냥~
</StepCard>

<Callout type="important">
  **가장 큰 차이점 한 줄 정리!**

  - 제미나이 **앱** → **사람**이 사용하는 도구
  - 제미나이 **API** → **내가 만든 프로그램(코드)**이 사용하는 도구
</Callout>

### 🛠️ 오늘 만들 것: 날씨 정보 앱

<StepCard step="목표" title="API 두 개를 연결해서 간단한 정보 앱을 만들어보자냥!">

**앱이 하는 일**

1. **날씨 API**(또는 공공데이터 API)에서 → 오늘의 날씨 정보를 가져온다냥
2. 그 정보를 **제미나이 API**에 전달해서 → 재미있는 날씨 코멘트를 받아온다냥
3. 결과를 → 화면에 보여준다냥

예를 들면 이런 결과가 나온다냥:

```python
🌤️ 오늘 서울 날씨: 맑음, 28도

🐱 코딩냥의 한마디:
"오늘은 햇살이 쨍쨍하다냥! 자외선 차단제 꼭 바르고,
시원한 물 많이 마시는 거 잊지 말라냥~"
```

</StepCard>

<Callout type="tip">
  **📦 사용할 재료 (API)**

  | 재료                                      | 역할                 | 비고                      |
  | ---                                       | ---                  | ---                       |
  | 날씨 API (OpenWeatherMap 또는 기상청 API) | 날씨 데이터 가져오기 | 선생님이 미리 키 발급해둠 |
  | 제미나이 API                              | AI 코멘트 생성하기   | 선생님이 미리 키 발급해둠 |

  API 키(Key)는 API를 쓸 수 있는 일종의 "회원증"이라고 생각하면 된다냥!
</Callout>

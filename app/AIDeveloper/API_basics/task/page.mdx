<MissionCard number={1} title="날씨 API 연결하기 " time="8~10분">

먼저 OpenWeatherMap API에 요청을 보내서 날씨 데이터를 받아오자냥. 아래 코드에서 **빈칸(`____`, `___`)을 직접 채워보자냥.**

[사용 방법 확인하기](https://openweathermap.org/api/current?collection=current_forecast)

```python
import requests

weather_api_key = "여기에 날씨 API 키 입력"
city = "Seoul"

url = f"https://api.openweathermap.org/data/2.5/weather?q={____}&appid={____}&units=metric"

response = requests.get(url)
data = response.json()

temperature = data["main"]["___"]       # 온도 키를 찾아보세요
description = data["weather"][0]["___"] # 날씨 설명 키를 찾아보세요

print(f"도시: {city}")
print(f"온도: {temperature}도")
print(f"날씨: {description}")
```

> **확인해보자냥 (빈칸 힌트)**
> - 첫 번째 `{____}` 자리에는 **도시 이름**을 담은 변수가 들어간다냥.
> - 두 번째 `{____}` 자리에는 **API 키**를 담은 변수가 들어간다냥.
> - `data["main"]["___"]` 에서 온도를 나타내는 키는 영어로 "온도"를 뜻하는 단어다냥
> - `data["weather"][0]["___"]` 에서 날씨 설명을 나타내는 키는 영어로 "설명"을 뜻하는 단어다냥.

**직접 해보기**
`city` 를 `Busan`, `Tokyo`, `New York` 등으로 바꿔가며 실행해보자냥. 각 도시의 날씨가 다르게 나오냥?

<MissionCard number={2} title="제미나이 API로 코멘트 만들기 " time="8~10분">

이번엔 제미나이 API에 날씨 데이터를 전달해서 AI 코멘트를 받아오자냥! 아래 코드에서 **빈칸(`____`)을 직접 채워보자냥.**


[제미나이 API 사용방법](https://ai.google.dev/gemini-api/docs/quickstart?hl=ko)


```python
from google import genai

client = genai.Client(api_key="여기에 제미나이 API 키 입력")

# 미션 1에서 가져온 날씨 데이터를 활용하자냥
temperature = 28   # 미션 1 결과로 바꿔보자냥
description = "clear sky"

prompt = f"""
오늘 {____}의 날씨는 {____}이고, 기온은 {____}도야.
이 날씨에 어울리는 짧고 재미있는 코멘트를 2~3문장으로 써줘.
친근하고 유머러스한 말투로 부탁해!
"""

response = client.models.generate_content(model="gemini-3.5-flash", contents=prompt)
print(response.text)
```

> **확인해보자냥 (빈칸 힌트)**
> - 첫 번째 `{____}` 자리에는 **도시 이름**이 들어간다냥.
> - 두 번째 `{____}` 자리에는 **날씨 설명** 변수가 들어간다냥.
> - 세 번째 `{____}` 자리에는 **온도** 변수가 들어간다냥.

<Callout type="tip">
  **프롬프트를 조금만 바꿔보자냥!** "친근하고 유머러스한" 부분을 "격식체로", "영어로" 처럼 다르게 바꾸면 결과가 어떻게 달라지는지 실험해보자냥.
</Callout>

</MissionCard>

<MissionCard number={3} title="두 API 합쳐서 완성하기 " time="8~10분">

이제 미션 1과 미션 2를 하나로 합쳐서 날씨 정보 앱을 완성해보자냥! 아래 코드에서 **빈칸(`____`)을 채우며 완성해보자냥.**

```python
import requests
from google import genai

# --- API 키 설정 ---
weather_api_key = "여기에 날씨 API 키 입력"
gemini_api_key  = "여기에 제미나이 API 키 입력"

# --- 날씨 데이터 가져오기 ---
city = "Seoul"
url = f"https://api.openweathermap.org/data/2.5/weather?q={city}&appid={weather_api_key}&units=metric"
data = requests.get(url).json()

temperature = data["main"]["temp"]
description = data["weather"][0]["description"]

# --- 제미나이 AI 코멘트 만들기 ---
client = genai.Client(api_key=____)

prompt = f"오늘 {city} 날씨는 {description}이고 기온은 {temperature}도야. 재미있는 날씨 코멘트 2~3문장 써줘."
comment = client.models.generate_content(model="gemini-3.5-flash", contents=____).text

# --- 결과 출력 ---
print(f"🌤️ 오늘 {city} 날씨: {description}, {temperature}도")
print(f"\n🐱 코딩냥의 한마디:\n{____}")
```

> **확인해보자냥 (빈칸 힌트)**
> - `genai.Client(api_key=____)` 자리에는 제미나이 API 키 변수가 들어간다냥.
> - `client.models.generate_content(model="gemini-3.5-flash", contents=____)` 자리에는 위에서 만든 `prompt` 변수가 들어간다냥.
> - 마지막 `print` 에서 `{____}` 자리에는 AI 코멘트를 담은 변수가 들어간다냥.

### 핵심 비유 · 두 가게를 연결하는 심부름꾼

<Callout type="analogy">
  날씨 API는 **날씨 정보를 파는 가게**다냥. 제미나이 API는 **글을 써주는 작가 가게**다냥.

  내 코드는 **심부름꾼** 역할을 한다냥! 날씨 가게에서 정보를 사 온 다음, 그 정보를 들고
  작가 가게로 가서 "이 날씨에 맞는 코멘트 써줘"라고 부탁하는 거다냥. 두 가게를 내 코드가
  연결해주는 게 핵심이다냥!

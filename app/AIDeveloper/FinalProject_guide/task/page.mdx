  한 번에 다 하는 게 아니라, 하루에 한두 미션씩 차근차근 밟아가면 된다냥.

<MissionCard number={1} title="아이디어 씨앗 고르기 (23차시)" time="25~30분">

평소에 **불편했던 일**이나 **관심 있는 분야**를 떠올리며, 주제 후보 3개를 적어보자냥. 각 아이디어마다 "어떤 AI 기능이 필요할지"도 함께 메모하면 좋다냥!

아래 브레인스토밍 시트의 빈칸을 채워보자냥.

```
1. 평소에 불편하거나 시간이 많이 걸리는 일은 무엇인가요?
   -

2. 내가 관심 있는 분야/주제는 무엇인가요?
   -

3. 위 내용을 바탕으로 떠올린 아이디어 3가지:
   아이디어 A: ____________  (필요한 AI 기능: ____________)
   아이디어 B: ____________  (필요한 AI 기능: ____________)
   아이디어 C: ____________  (필요한 AI 기능: ____________)

4. 짝의 피드백:
   - 가장 흥미로운 아이디어: ____________
   - 이유: ____________
```

> **이번 미션의 목표**
> - 주제 후보 3개를 적었냥?
> - 짝과 아이디어를 나누고, 다음 시간까지 **주제 1개로 좁혔냥?**

</MissionCard>

<MissionCard number={2} title="기획서 = 설계도 완성하기 (24차시)" time="25~30분">

만들기 전에 **무엇을, 어떻게 만들지** 정리해두면 시간을 크게 아낄 수 있다냥. 14차시에서 블로그 글을 쓸 때도 "설계도(구조)"부터 만들었던 거 기억나냥? 앱도 똑같다냥!

아래 기획서 빈칸을 채워, 내 앱의 설계도를 완성해보자냥.

```
■ 앱 이름:

■ 한 줄 설명 (이 앱은 ___________________ 을 위한 앱입니다):


■ 주요 사용자:

■ 핵심 기능 (1~3개, 우선순위 순):
  1.
  2.
  3.

■ 사용할 기술 스택 (체크):
  [ ] 제미나이 API (텍스트 생성)
  [ ] 이미지 인식 / 이미지 생성
  [ ] LangChain (도구 연결)
  [ ] RAG (문서 기반 검색)
  [ ] Streamlit (화면)
  [ ] 텔레그램 봇
  [ ] 기타:

■ 화면 구상 (간단한 스케치 또는 설명):


■ 예상되는 어려움과 대비책:

```

<Callout type="important">
  **핵심 기능은 1~3개로!** 욕심내서 기능을 너무 많이 넣으면 완성을 못 한다냥.
  모둠 친구·선생님 피드백을 받아 너무 크면 줄이고, 너무 작으면 살짝 키워보자냥.
</Callout>

</MissionCard>

<MissionCard number={3} title="대표 메뉴부터! 핵심 기능 구현 (25차시)" time="25~30분">

가게를 열 때 모든 메뉴를 동시에 만들기보다, **대표 메뉴 하나**를 먼저 완성하는 게 좋다냥. 오늘은 기획서의 **핵심 기능 1개**를 `.ipynb`에서 먼저 만들어보자냥.

- 프롬프트 작성, API 호출 등 가장 중요한 기능부터 구현한다냥.
- `.ipynb`의 장점을 살려, **작은 단위(셀)로 자주 실행**하며 확인한다냥.
- 막히면 지난 차시 코드를 **참고 템플릿**으로 가져다 써도 좋다냥!

> **이번 미션의 목표**
> - 핵심 기능 1개가 `.ipynb`에서 동작하냥?
> - 짝에게 시연해보고, 다음에 만들 기능을 메모했냥?

</MissionCard>

<MissionCard number={4} title="진열대에 올리기 — UI 연결 또는 기능 확장 (26차시)" time="25~30분">

맛있는 메뉴(핵심 기능)를 만들었다면, 이제 손님이 볼 수 있는 **진열대(화면)**에 올릴 차례다냥. 내 진도에 맞게 둘 중 하나를 골라 진행하면 된다냥!

<Callout type="tip">
  **두 갈래 길 — 내 상황에 맞게 고르자냥**

  - **UI 연결 그룹** — 핵심 기능을 Streamlit 화면(`st.text_input`, `st.button` 등)에 연결한다냥.
  - **기능 확장 그룹** — 기획서의 **두 번째 기능** 구현을 시작한다냥.
</Callout>

> **자가 점검**
> - "오늘 목표 중 어디까지 왔냥?"
> - 막힌 부분은 메모해뒀냥? (다음 시간이나 점검 때 질문하면 된다냥!)

</MissionCard>

<MissionCard number={5} title="주방 시스템 점검 — 통합 및 디버깅 (27차시)" time="25~30분">

여러 코너에서 만든 메뉴를 한 주방에서 함께 운영하면 동선이 엉킬 수 있다냥. 오늘은 만든 기능들을 **하나의 앱으로 통합**하고, 전체 흐름을 테스트하는 날이다냥!

- 지금까지 만든 기능들을 하나의 앱으로 합친다냥.
- 통합 후 **전체 흐름을 테스트**하고, 발견한 오류를 수정한다냥.

<Callout type="important">
  **통합할 때 자주 생기는 문제들** — 변수명 충돌, 실행 순서 꼬임 등이 흔하다냥.
  디버깅 팁을 떠올리자냥: **에러 메시지를 끝까지 읽기**, **작은 단위로 나눠 테스트하기!**
</Callout>

> **이번 미션의 목표**
> - 기능들이 하나의 앱에서 함께 잘 돌아가냥?
> - 내일 중간 점검에서 받고 싶은 피드백 포인트를 메모했냥?

</MissionCard>

<MissionCard number={6} title="오픈 전 시식회 — 동료 피드백 (28차시)" time="25~30분">

정식으로 가게를 열기 전, 친구들에게 미리 맛을 보여주고 의견을 듣는 **시식회**다냥! 피드백은 비판이 아니라 **"더 좋아지기 위한 발견"**이라는 걸 꼭 기억하자냥.

모둠을 돌며 서로의 앱을 직접 써본 뒤(갤러리 워크), 아래 시트를 작성해보자냥. 피드백은 **1-1-1 형식**(좋은 점 1개 · 개선점 1개 · 질문 1개)이 좋다냥.

```
■ 평가한 앱 이름:

■ 좋았던 점 (1개 이상):


■ 더 좋아지면 좋을 점 (1개 이상):


■ 궁금한 점/질문:


■ (작성자에게) 받은 피드백 중 반영하고 싶은 것:

```

> **이번 미션의 목표**
> - 가장 유용했던 피드백 1개를 골랐냥?
> - 받은 의견을 바탕으로 남은 시간의 **우선순위를 다시 정리**했냥?

</MissionCard>

<MissionCard number={7} title="마지막 플레이팅 — 피드백 반영 및 마무리 (29차시)" time="25~30분">

요리는 거의 완성됐다냥. 이제 마지막으로 접시에 예쁘게 담는 시간이다냥. 작은 정리와 마무리가 전체 인상을 크게 바꿀 수 있다냥!

- **우선순위가 높은 피드백부터** 반영해 기능을 보완/수정한다냥.
- 여러 입력값으로 **최종 동작 테스트**를 한다냥.
- 21~22차시에서 만든 **윤리 가이드라인** 항목을 점검한다냥. (예: "AI가 만든 결과임을 표시했는가?")
- 최종본을 **저장/백업**하고, 발표에서 강조하고 싶은 부분을 메모한다냥.

<Callout type="tip">
  **완벽하지 않아도 괜찮다냥!** 발표는 완성품 자랑이 아니라, **배운 과정**을
  보여주는 자리라냥. 지금 모습 그대로도 충분히 멋지다냥!
</Callout>

</MissionCard>

<MissionCard number={8} title="개업식 초청장 준비 — 발표 준비 (30차시)" time="25~30분">

가게를 열기 전, 손님들에게 무엇을 어떤 순서로 보여줄지 정하는 시간이다냥. 너무 많은 걸 보여주려 하지 말고, **가장 자신 있는 부분**을 잘 보여주는 게 중요하다냥!

아래 발표 구성 시트를 채우고, 짝과 **리허설**까지 해보자냥.

```
■ 1. 어떤 문제를 해결하고 싶었나요? (30초)


■ 2. 어떻게 만들었나요? (사용한 기술, 1분)
   -

■ 3. 시연 계획 (1~2분)
   - 보여줄 입력값/예시:
   - 시연 순서:

■ 4. 배운 점 / 어려웠던 점 (1분)
   -

■ 예상 질문과 답변 준비:
   Q:
   A:
```

<Callout type="important">
  **시연 실패를 줄이는 꿀팁!** 시연 때 보여줄 **입력값을 미리 정해두자냥.**
  잘 동작하는 게 확인된 예시를 준비해두면, 발표 중에 당황할 일이 줄어든다냥!
</Callout>

</MissionCard>

<MissionCard number={9} title="가게 개업식 — 발표하기 (31~32차시)" time="1인 5분 내외">

드디어 그동안 준비한 가게의 **문을 여는 날**이다냥! 내가 만든 것을 친구들에게 보여주고, 다른 친구들의 가게도 둘러보며 많이 배우는 시간이다냥.

- **발표할 때** — 준비한 4가지(문제 → 방법 → 시연 → 배운 점) 순서대로 차분하게 보여주자냥.
- **들을 때** — 경청하고, 박수 쳐주고, **질문 1개씩** 준비해두자냥.
- 발표를 들으며 인상 깊었던 점을 메모해두면 회고에 도움이 된다냥.

> 떨려도 괜찮다냥! 여기까지 온 것만으로도 정말 대단한 거다냥. 자신 있게 가보자냥!

</MissionCard>

## 최종 제출 체크리스트

<StepCard step="제출" title="발표 전, 이것만은 꼭 확인하자냥!">

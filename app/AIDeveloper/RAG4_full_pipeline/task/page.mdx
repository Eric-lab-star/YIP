<MissionCard number={1} title="실제 문서 파일을 벡터DB에 넣기 " time="8~10분">

지금까지는 코드에 직접 적어둔 짧은 예시 문장으로 연습했다냥. 오늘은 드디어 **진짜 문서 파일**을 통째로 읽어서 벡터DB에 넣어볼 거다냥!

먼저 실습용 문서를 준비하자냥. 아래 예시를 복사해서 **`school_guide.txt`** 라는 이름으로, 코드 파일(`.py` 또는 `.ipynb`)과 **같은 폴더**에 저장하면 된다냥. (학교 가정통신문, 동아리 소개글처럼 **내가 가진 문서**를 써도 좋다냥!)

```text
[코딩냥 중학교 학교 생활 안내]

1. 등교 시간
등교 시간은 오전 8시 30분까지입니다. 정문은 오전 7시 30분에 열리며,
8시 40분 이후에 도착하면 지각으로 처리됩니다.

2. 점심시간
점심시간은 오후 12시 30분부터 1시 20분까지입니다. 급식실은 본관 1층에
있으며, 학년별로 정해진 순서에 따라 이용합니다. 식단표는 매주 월요일
학교 홈페이지에 올라옵니다.

3. 도서관 이용
도서관은 평일 오전 9시부터 오후 5시까지 이용할 수 있습니다. 책은
1인당 3권까지, 2주 동안 빌릴 수 있습니다. 시험 기간에는 오후 7시까지
연장 운영합니다.

4. 동아리 활동
동아리 활동은 매주 수요일 5~6교시에 진행됩니다. 코딩 동아리, 방송
동아리, 밴드부, 축구부 등 20여 개의 동아리가 있으며, 매년 3월에
신입 부원을 모집합니다.
```

이제 이 파일을 **읽고 → 청킹으로 자르고 → 벡터DB에 저장**까지 한 번에 해보자냥. 청킹 함수는 2차시에 우리가 직접 만든 `split_into_chunks`를 그대로 다시 쓰는 게 포인트라냥! 아래 템플릿에서 **빈칸(`____`)을 채워보자냥.**

```python
import chromadb

# 1) 문서 파일을 열어서 통째로 읽기
with open(____, "r", encoding="utf-8") as f:
    document = f.read()

print(f"문서 길이: {len(document)}자")

# 2) 2차시에 만든 청킹 함수로 잘게 나누기
def split_into_chunks(text, chunk_size=200):
    chunks = []
    for i in range(0, len(text), chunk_size):
        chunks.append(text[i:i + chunk_size])
    return chunks

chunks = split_into_chunks(____, chunk_size=200)
print(f"총 {len(chunks)}개의 청크로 나뉘었습니다.")

# 3) 벡터DB(서랍장)에 저장하기
client = chromadb.Client()
collection = client.get_or_create_collection(name="my_documents")

ids = [f"chunk_{i}" for i in range(len(chunks))]
collection.add(documents=chunks, ids=ids)

print(f"{collection.count()}개의 청크가 저장되었습니다.")
```

> **확인해보자냥 (빈칸 힌트)**
> - `open(____, ...)` 자리에는 방금 저장한 **파일 이름**을 문자열로 넣으면 된다냥. 예: `"school_guide.txt"`
> - `split_into_chunks(____, ...)` 자리에는 파일에서 읽어온 텍스트가 담긴 변수를 넣으면 된다냥. `f.read()`의 결과를 어디에 담아뒀냥?
> - 문서 길이 · 청크 개수 · 저장 개수가 차례대로 잘 출력되냥?

<Callout type="important">
  **`FileNotFoundError`(파일을 찾을 수 없음) 에러가 난다면?**

  `.txt` 파일이 코드 파일과 **같은 폴더**에 있는지 먼저 확인하자냥.
  구글 코랩(Colab)을 쓴다면 왼쪽 폴더 아이콘을 눌러 파일을 **업로드**하면 된다냥.
</Callout>

<Callout type="tip">
  이렇게 문서를 **읽고 → 자르고 → 저장하는** 준비 과정을 **인덱싱(indexing)**이라고 부른다냥.
  문서가 바뀌지 않는 한 **한 번만** 해두면 되고, 그 다음부터는 질문할 때마다 벡터DB에서 바로바로 꺼내 쓸 수 있다냥!
</Callout>

<MissionCard number={2} title="RAG 함수 만들기 " time="8~10분">

검색(벡터DB)과 생성(제미나이)을 한 함수 안에서 합쳐볼 거다냥! 아래 템플릿에서 **빈칸(`____`)을 직접 채워보자냥.**

특히 prompt 안의 **역할** 빈칸이 중요하다냥. 지난 시간에 배운 "좋은 프롬프트 작성법"을 떠올리며 채워보자냥!

```python
from google import genai

genai_client = genai.Client(api_key="여기에 본인의 제미나이 API 키 입력")

# 미션 1에서 문서를 저장해둔 collection을 그대로 이어서 사용한다냥

def rag_answer(question):
    # 1) 벡터DB에서 관련 청크 검색
    results = collection.query(query_texts=[question], n_results=2)
    retrieved_chunks = results["documents"][0]

    # 2) 검색된 내용을 하나의 문자열로 합치기
    context = "\n".join(retrieved_chunks)

    # 3) 역할/맥락/지시/형식을 담은 prompt를 직접 작성해보세요
    prompt = f"""
    너는 {____} 역할을 맡은 AI 도우미야.
    아래 [참고 자료]만을 근거로 질문에 답해줘.
    참고 자료에 없는 내용이면 "자료에서 찾을 수 없습니다"라고 답해줘.

    [참고 자료]
    {context}

    [질문]
    {question}
    """

    response = genai_client.models.generate_content(model="gemini-3.5-flash", contents=prompt)
    return response.text

# 질문을 직접 작성해 테스트해보세요
question = ____
print(rag_answer(question))
```

> **확인해보자냥 (빈칸 힌트)**
> - `너는 {____} 역할을` 자리에는 어떤 도우미인지 적으면 된다냥. 예: `"친절한 도서관 사서"`, `"우리 학교 안내 도우미"` 처럼!
> - `question = ____` 자리에는 **미션 1에서 넣은 문서에 있는 내용**을 질문으로 넣어보자냥. 예: `question = "우리 학교 점심시간은 언제야?"`

<MissionCard number={3} title="다양한 질문으로 테스트하기 " time="8~10분">

이번 미션이 RAG의 진짜 가치를 느끼는 순간이라냥! 미션 1에서 넣은 **진짜 문서**를 상대로, 두 종류의 질문을 던져서 결과를 비교해보자냥.

<StepCard step="실험" title="이렇게 테스트해보자냥">

1. **문서에 있는 내용**을 질문하기 → 자료를 바탕으로 친절하게 답하는지 확인하기 (예: "도서관에서 책을 몇 권까지 빌릴 수 있어?")
2. **문서에 없는 내용**을 일부러 질문하기 → "자료에서 찾을 수 없습니다"라고 솔직하게 답하는지 확인하기 (예: "교장 선생님 성함이 뭐야?")

</StepCard>

> **확인해보자냥**
> - 문서에 있는 질문에는 자료 내용을 근거로 답하냥?
> - 문서에 없는 질문에는 아는 척하지 않고 "찾을 수 없습니다"라고 답하냥?

<Callout type="tip">
  **재미있는 비교 실험!**

  prompt에서 `참고 자료에 없는 내용이면 "자료에서 찾을 수 없습니다"라고 답해줘.` 이 한 줄을 **넣었을 때**와 **뺐을 때**의 결과를 비교해보자냥.

  이 줄을 빼면 AI가 자료에 없는 내용도 그럴듯하게 지어내기도 한다냥(이걸 환각이라고 부른다냥!). RAG에서 prompt 한 줄이 얼마나 중요한지 직접 느껴보자냥!
</Callout>

<MissionCard number={4} title="Streamlit으로 RAG 챗봇 완성하기 " time="8~10분">

마지막이다냥! 미션 2에서 만든 `rag_answer` 함수를 Streamlit 화면에 연결해서, 입력창에 질문을 받아 답을 보여주는 **진짜 RAG 챗봇**을 완성해보자냥!

아래 템플릿에서 **빈칸(`____`)을 채워보자냥.**

```python
import streamlit as st

st.title("나만의 RAG 챗봇")
st.write("문서 자료를 바탕으로 질문에 답하는 AI입니다.")

user_question = st.text_input("질문을 입력하세요")

if st.button("질문하기"):
    answer = ____  # rag_answer 함수를 호출해보세요
    st.write("### 답변")
    st.write(answer)
    
    # (선택) 참고한 자료도 함께 보여주기
    with st.expander("참고한 자료 보기"):
        results = collection.query(query_texts=[user_question], n_results=2)
        for doc in results["documents"][0]:
            st.write("-", doc)
```

> **확인해보자냥 (빈칸 힌트)**
> - `answer = ____` 자리에는 미션 2에서 만든 함수를 호출하면 된다냥. 입력창의 값 `user_question`을 넘겨주면 된다냥!
> - 정답이 떠오르냥? `rag_answer(user_question)` 처럼 쓰면 된다냥.

**직접 해보기**

터미널에서 `streamlit run app.py`로 실행하고, 입력창에 여러 질문을 넣어보자냥. "참고한 자료 보기"를 펼쳐서, AI가 어떤 자료를 보고 답했는지도 구경해보자냥!

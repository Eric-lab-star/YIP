## 오늘의 학습 목표

<Callout type="goal">
  - 검색(**Retrieval**)과 생성(**Generation**)을 연결하는 전체 **RAG 흐름**을 이해한다.
  - 지난 3차시 동안 만든 결과물(**임베딩 · 청킹 · 벡터DB**)을 하나의 앱으로 통합한다.
  - 검색 결과를 prompt에 잘 넣는 것이 답변 품질을 좌우한다는 걸 안다.
</Callout>

## 2단계 · 핵심 비유 — "AI 사서의 완성" (3분)

<NyangSpeech>
  RAG가 뭔지 한 번에 와닿게 해줄 비유가 있다냥. 도서관을 한번 떠올려보자냥!
</NyangSpeech>

지금까지 우리는 **도서관(벡터DB)**에서 관련 자료를 빠르게 찾아내는 시스템을 만들었다냥. 그런데 손님이 이렇게 말하면 어떨까냥?

> "이 자료들 중에서 **내 질문에 맞는 답**을 정리해서 설명해주세요!"

자료를 찾는 것과, 그 자료를 **읽고 정리해서 설명하는 것**은 다른 일이다냥. 그래서 오늘은 두 명의 전문가를 한 팀으로 묶는다냥.

### 질문이 답변이 되는 네 정거장

<StepCard step="흐름" title="RAG 파이프라인 한눈에 보기">

1. **사용자 질문** — 손님이 궁금한 걸 물어본다냥.
2. **검색 (Retrieval)** — 벡터DB에서 질문과 **관련된 청크**를 찾아온다냥.
3. **prompt 구성** — 검색 결과 + 질문을 하나의 **prompt**로 합친다냥.
4. **생성 (Generation)** — 제미나이가 그 prompt를 읽고 **답변을 생성**한다냥!

</StepCard>

<Callout type="important">
  여기서 정말 중요한 점이 있다냥! **검색 결과를 prompt에 어떻게 넣느냐**가 답변 품질을 크게 좌우한다냥.
  지난 시간에 배운 **"좋은 프롬프트 작성법"**이 바로 여기서 빛을 발한다냥. 역할 · 맥락 · 지시 · 형식을 잘 담아야 좋은 답이 나온다냥!
</Callout>

### RAG의 핵심 한 문장

<Callout type="tip">
  RAG의 핵심은 이거다냥.

  **"AI가 원래 모르는 내용도, 검색해온 자료를 참고하면 답할 수 있다!"**

  제미나이는 우리 학교 자료나 내 개인 문서를 학습한 적이 없지만,
  그 자료를 찾아서 prompt에 넣어주면 마치 아는 것처럼 답할 수 있게 된다냥. 신기하지 않냥?
</Callout>

## 4단계 · 코드로 미리 보는 RAG 함수 (3분)

<NyangSpeech>
  실습에서 직접 빈칸을 채워볼 거지만, 먼저 RAG 함수가 어떻게 생겼는지
  구경만 해보자냥. 위에서 배운 4단계가 코드에 그대로 들어 있다냥!
</NyangSpeech>

```python
import chromadb
from google import genai

genai_client = genai.Client(api_key="여기에 본인의 제미나이 API 키 입력")

# 지난 시간 만든 Chroma DB와 collection을 그대로 사용
client = chromadb.Client()
collection = client.get_or_create_collection(name="my_documents")

def rag_answer(question):
    # 1) 벡터DB에서 관련 청크 검색 (Retrieval)
    results = collection.query(query_texts=[question], n_results=2)
    retrieved_chunks = results["documents"][0]

    # 2) 검색된 내용을 하나의 문자열로 합치기
    context = "\n".join(retrieved_chunks)

    # 3) 검색 결과 + 질문을 prompt로 구성
    prompt = f"""
    아래 [참고 자료]만을 근거로 질문에 답해줘.

    [참고 자료]
    {context}

    [질문]
    {question}
    """

    # 4) 제미나이가 답변 생성 (Generation)
    response = genai_client.models.generate_content(model="gemini-3.5-flash", contents=prompt)
    return response.text
```

보이냥? `collection.query(...)`가 **검색**, `genai_client.models.generate_content(...)`가 **생성**이라냥. 그 사이에서 검색 결과를 prompt에 끼워넣는 게 RAG의 핵심이라냥!

<Callout type="note">
  걱정 말라냥! 실습 페이지에서는 prompt 안의 **역할**과 **테스트 질문**을 직접 채워볼 거다냥. 지금은 "아, 이렇게 생겼구나" 정도만 느끼면 충분하다냥.
</Callout>

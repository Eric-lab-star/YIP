<MissionCard number={1} title="수학 문제 사진 준비하기 📷" time="5분">

먼저 AI에게 보여줄 **수학 문제 사진**을 준비하자냥. 직접 촬영하거나, 선생님이 나눠준 예시 이미지를 써도 좋다냥.

준비한 이미지 파일을 코드와 **같은 폴더**에 두고, 파일 이름을 기억해두자냥. (예: `math_problem.jpg`)

> ✅ **확인해보자냥**

<Callout type="tip">
  🐾 **실험 꿀팁** — 깨끗하게 찍은 사진과 일부러 흐릿하게 찍은 사진을 **둘 다** 준비해보자냥. 나중에 결과를 비교하면, 입력 품질이 얼마나 중요한지 직접 체감할 수 있다냥!

<MissionCard number={2} title="사진 속 문제 인식하기 👀" time="10~12분">

이미지를 제미나이에 전달해서, **"무슨 문제인지" 글자로 읽어보게** 만들어볼 거다냥.

아래 템플릿에서 **빈칸(`____`)을 직접 채워보자냥.**

```python
import google.generativeai as genai
import PIL.Image

genai.configure(api_key="여기에 본인의 제미나이 API 키 입력")
model = genai.GenerativeModel("gemini-1.5-flash")

# 수학 문제 사진 불러오기
image = PIL.Image.open("math_problem.jpg")

# 이미지와 함께 보낼 지시문을 직접 작성해보세요
read_prompt = ____  # 예: "이 사진 속에 있는 수학 문제를 글자로 그대로 읽어줘."

response = model.generate_content([read_prompt, image])
print("인식된 문제:", response.text)
```

> ✅ **확인해보자냥 (빈칸 힌트)**
> - `read_prompt = ____` 자리에는 AI에게 **사진을 어떻게 읽어달라**고 부탁하는 문장을 넣으면 된다냥.
> - 가장 쉽게는 `read_prompt = "이 사진 속에 있는 수학 문제를 글자로 그대로 읽어줘."` 처럼 따옴표로 감싼 문장을 넣으면 된다냥!

✍️ **직접 해보기**
실행해서, AI가 사진 속 문제를 글자로 잘 읽어내는지 확인해보자냥. 또렷한 사진과 흐릿한 사진을 **둘 다** 넣어보고 결과가 어떻게 다른지 비교해보자냥!

</MissionCard>

<MissionCard number={3} title="단계별 풀이 생성하기 🧠" time="10~13분">

이번엔 사진 속 문제를 **단계별로 풀어주는** 프롬프트를 완성해볼 거다냥. 답만 툭 던지는 게 아니라, **풀이 과정 → 정답** 순서로 차근차근 설명하게 만드는 게 핵심이다냥!

아래 템플릿에서 **빈칸(`{____}`)을 직접 채워보자냥.**

```python
# 풀이용 프롬프트를 직접 완성해보세요
solve_prompt = """
너는 친절한 수학 선생님이야.
이 사진 속 수학 문제를 풀어줘. 단,
1) 먼저 문제가 무엇을 묻는지 설명하고,
2) {____} (단계별 풀이 과정을 어떻게 보여줄지 지시)
3) 마지막에 '정답: '으로 정답을 명확히 알려줘.
중학생도 이해할 수 있게 쉽게 설명해줘.
"""

response = model.generate_content([solve_prompt, image])
print(response.text)
```

> ✅ **확인해보자냥 (빈칸 힌트)**
> - `{____}` 자리에는 AI가 **풀이 과정을 어떻게 보여줄지** 지시하는 문장을 넣으면 된다냥.
> - 예를 들어 `풀이 과정을 한 단계씩 순서대로 자세히 보여주고,` 처럼 적으면 좋다냥!

### 🧑‍🏫 핵심 비유 — "풀이 과정을 꼭 보여주는 선생님"

<StepCard step="비유" title="왜 단계별 풀이를 부탁할까냥?">

좋은 수학 선생님은 **답만** 알려주지 않고, "왜 이렇게 푸는지" 과정을 차근차근 보여준다냥.

AI에게도 **"정답만 말하지 말고, 단계별로 풀이를 보여줘"**라고 부탁하는 게 중요하다냥. 그래야 우리가 답을 **외우는** 게 아니라 **푸는 방법**을 배울 수 있으니까냥!

프롬프트에 **"단계별로"**, **"쉽게 설명"**이라는 조건을 넣는 이유가 바로 이것이다냥 🐟

</StepCard>

✍️ **직접 해보기**
간단한 사칙연산부터 방정식까지, **여러 난이도**의 문제 사진으로 테스트해보자냥. 풀이가 단계별로 잘 나오냥?

<MissionCard number={1} title="나쁜 프롬프트 vs 좋은 프롬프트 비교 실험" time="8~10분">

같은 주제(여행 추천)를 두고, **단순한 프롬프트**와 **역할·맥락·지시·형식을 갖춘 프롬프트**의 결과를 나란히 출력해서 비교해보자냥!

아래 템플릿에서 **빈칸(`____`)을 직접 채워보자냥.**

```python
from langchain_google_genai import ChatGoogleGenerativeAI

llm = ChatGoogleGenerativeAI(
    model="gemini-3.5-flash",
    google_api_key="여기에 본인의 제미나이 API 키 입력"
)

# 나쁜 프롬프트 (정보가 부족함)
bad_prompt = "여행지 추천해줘"

# 좋은 프롬프트 - 빈칸을 채워 역할/맥락/지시/형식을 추가해보세요
good_prompt = f"""
너는 {____} 역할을 맡은 여행 전문가야.
{____}을(를) 좋아하는 사람에게,
{____} 조건에 맞는 국내 여행지 3곳을
{____} 형식으로 추천해줘.
"""

print("=== 나쁜 프롬프트 결과 ===")
print(llm.invoke(bad_prompt).content)

print("\n=== 좋은 프롬프트 결과 ===")
print(llm.invoke(good_prompt).content)
```

> **확인해보자냥 (빈칸 힌트)**
> - 첫 번째 `{____}` → **역할**: 어떤 전문가일까냥? (예: "친절한", "20년 경력의")
> - 두 번째 `{____}` → **맥락**: 무엇을 좋아하는 사람이냥? (예: "바다", "맛집")
> - 세 번째 `{____}` → **지시**: 어떤 조건이냥? (예: "예산 20만 원", "당일치기")
> - 네 번째 `{____}` → **형식**: 어떤 모양으로 받고 싶냥? (예: "표", "번호 목록")

**직접 해보기**
두 결과를 비교해보자냥. 좋은 프롬프트 쪽이 훨씬 또렷하고 원하는 모양으로 나오냥?

</MissionCard>

<Callout type="important">
  **"나쁜 프롬프트" 결과가 의외로 괜찮게 나올 수도 있다냥!**

  그럴 때는 **더 구체적인 조건**(예: 예산, 인원수, 계절)을 좋은 프롬프트에 추가해서 다시 비교해보자냥. 조건이 많아질수록 차이가 더 또렷하게 보인다냥!
</Callout>

<MissionCard number={2} title="프롬프트로 여섯 가지 일 시켜보기 + 제로샷 CoT" time="6~8분">

프롬프트만 바꿔서 AI에게 여러 가지 일을 시켜보자냥! 아래에서 마음에 드는 걸 **두세 개 골라** `llm.invoke()`로 돌려보자냥.

```python
# 1) 요약 — 긴 글을 한 문장으로
summ = """항생제는 세균 감염을 치료하는 약이야. 세균을 죽이거나 번식을 막아서
우리 몸의 면역계가 감염과 싸우게 도와줘. 다만 바이러스에는 효과가 없어.

위 내용을 한 문장으로 요약해줘."""

# 2) 질문 답변 — 맥락 안에서만 답하고, 모르면 "잘 모르겠어"
qa = """아래 맥락만 보고 답해줘. 확신이 없으면 "잘 모르겠어"라고만 답해.
맥락: OKT3는 처음에 생쥐에서 유래한 항체야.
질문: OKT3는 처음에 무엇에서 유래했어?"""

# 3) 분류 — 정해진 범주로
cls = """다음 문장의 감정을 중립/부정/긍정 중 하나로 분류해줘.
문장: 음식은 그냥 그랬어."""

for name, p in [("요약", summ), ("질문답변", qa), ("분류", cls)]:
    print(f"=== {name} ===")
    print(llm.invoke(p).content, "\n")
```

이번엔 **제로샷 CoT**의 마법을 확인해보자냥. 아래 문제를 두 번 물어보되, 한 번은 그냥, 한 번은 **"차근차근 생각해보자"**를 붙여서 결과를 비교하자냥!

```python
problem = "사과를 10개 샀어. 이웃에게 2개, 수리 기사에게 2개를 줬어. 5개를 더 사서 1개를 먹었어. 남은 사과는 몇 개야?"

print("=== 그냥 물어보기 ===")
print(llm.invoke(problem).content)

print("\n=== 차근차근 생각해보자 붙이기 ===")
print(llm.invoke(problem + "\n\n차근차근 생각해보자.").content)
```

> **확인해보자냥**
> - 요약/질문답변/분류가 각각 원하는 모양으로 나왔냥?
> - 질문 답변에서 맥락에 없는 걸 물으면 "잘 모르겠어"라고 답하냥? (일부러 엉뚱한 질문도 넣어보자냥!)
> - "차근차근 생각해보자" 한 문장을 붙였을 때 계산이 더 정확해졌냥?

</MissionCard>

<Callout type="tip">
  **어려운 문제일수록 "차근차근 생각해보자"가 힘을 발휘한다냥!** 예시를 몇 개 보여주는 **Few-shot**, 푸는 과정까지 보여주는 **CoT**도 같은 원리다냥 — AI에게 생각할 시간과 본보기를 주는 거다냥!
</Callout>

<MissionCard number={3} title="나만의 프롬프트 템플릿 만들기" time="8~10분">

이번엔 미션 1의 빈칸을 **나만의 답**으로 바꿔서, 세상에 하나뿐인 프롬프트를 만들어보자냥!

<StepCard step="템플릿" title="이 네 칸만 채우면 완성이다냥!">

| 요소 | 내가 채울 내용 (예시) |
|---|---|
| 역할(Role) | "너는 ○○ 전문가야" |
| 맥락(Context) | "○○한 사람에게 / ○○ 상황에서" |
| 지시(Instruction) | "○○을 ○개, ○○ 조건으로" |
| 형식(Format) | "표 / 목록 / 이모지 포함 등으로" |

</StepCard>

> **확인해보자냥**
> - 네 가지 요소가 **모두** 들어갔냥?
> - 친구의 템플릿과 비교해보자냥 — 같은 틀인데도 답이 다양하게 나오는 게 신기하지 않냥?

<Callout type="tip">
  **친구마다 다른 답을 넣어보자냥!** 같은 템플릿이라도 역할·맥락을 다르게 채우면 AI 결과가 완전히 달라진다냥. 여행 말고 음식, 운동, 공부 같은 다른 주제로도 바꿔보면 더 재밌다냥!
</Callout>

<MissionCard number={4} title="LangChain Agent에 적용하기 " time="8~10분">

이번엔 지난 시간에 만든 **Agent**에 오늘 배운 걸 적용해볼 거다냥! Tool의 `description`을 또렷하게 고치고, 역할·맥락·형식을 담은 질문을 넣어서 결과가 어떻게 달라지는지 확인해보자냥.

아래 템플릿에서 **빈칸(`____`)을 직접 채워보자냥.**

```python
from langchain.agents import create_agent
from langchain_core.tools import tool

# 지난 시간 만든 get_weather 함수에 @tool을 붙이고,
# 설명문(docstring)을 더 명확하게 수정해보세요
@tool
def get_weather(city: str) -> str:
    """____"""  # 어떤 입력을 받고, 어떤 정보를 돌려주는지 구체적으로 작성
    ...  # 지난 시간 만든 날씨 API 호출 코드

agent = create_agent(llm, [get_weather])

# 역할/맥락/형식을 담은 질문을 직접 작성해보세요
question = ____
result = agent.invoke({"messages": [{"role": "user", "content": question}]})
print(result["messages"][-1].content)
```

> **확인해보자냥 (빈칸 힌트)**
> - 함수의 **설명문(docstring)** `"""____"""` → 도구가 **어떤 입력을 받고, 어떤 정보를 돌려주는지** 구체적으로 적자냥. (예: "도시 이름을 입력받아 해당 도시의 현재 온도와 날씨 상태를 알려준다")
> - `question = ____` → 역할·맥락·형식을 담은 문장이다냥. (예: "너는 친절한 날씨 안내원이야. 서울의 오늘 날씨를 한 문장으로 알려줘")

</MissionCard>

<Callout type="important">
  **Agent의 `messages`(대화 과정)를 꼭 확인하자냥!**

  설명문(docstring)을 고친 뒤에는 `result["messages"]`를 하나씩 `pretty_print()`로 출력해서, **Agent의 판단 과정(어떤 도구를 왜 골랐는지)**이 이전과 어떻게 달라졌는지 비교해보자냥. 설명이 또렷해지니 Agent가 덜 헤맨다냥!
</Callout>

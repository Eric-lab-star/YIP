## 1단계 · 왜 똑같은 AI인데 결과가 다르냥? (5분)

<NyangSpeech>
  먼저 질문 하나 던질게냥. "왜 **똑같은 AI**인데, 설명을 어떻게 쓰느냐에 따라
  결과가 이렇게 달라질까냥?" 지난 시간을 떠올려보자냥.
</NyangSpeech>

지난 시간, Tool의 `description`(설명)이 모호하게 쓰여 있을 때, Agent는 "이 도구를 언제 써야 하지?" 하고 헷갈려서 엉뚱하게 행동했다냥. 반대로 설명을 또렷하게 적어줬더니 척척 알아서 도구를 골랐지?

<Callout type="important">
  ⚠️ **프롬프트는 "AI에게 명령하는 코드"와 같다냥.**

  코드에 오타가 있으면 에러가 나듯이, 프롬프트가 **모호하면 곧 엉뚱한 결과(오류)**로 이어진다냥. AI는 우리가 적어준 글자만큼만 이해한다냥!
</Callout>

## 2단계 · 핵심 비유: "같은 재료, 다른 주문서" (3분)

<NyangSpeech>
  이걸 식당으로 비유하면 단번에 이해된다냥. 잘 들어보라냥! 🍳
</NyangSpeech>

요리사(AI)에게 그냥 **"맛있게 해주세요"**라고만 말하면, 요리사는 무엇을 만들어야 할지 막막하다냥. 한식? 양식? 매운 거? 단 거? 알 수가 없다냥.

하지만 **"한식을 좋아하는 손님을 위해, 맵지 않게, 15분 안에 만들 수 있는 점심 메뉴를 추천해주세요"**라고 말하면, 요리사는 훨씬 정확하고 만족스러운 요리를 만들어낸다냥!

<Callout type="analogy">
  **비유로 이해하기 — 주문서가 곧 프롬프트**

  | 식당 | 우리 AI |
  |---|---|
  | 요리사 | AI(LLM) |
  | 손님이 건넨 주문서 | 우리가 적은 프롬프트(prompt) |
  | 대충 쓴 주문서 → 막막한 요리 | 모호한 프롬프트 → 엉뚱한 답변 |
  | 자세히 쓴 주문서 → 만족스러운 요리 | 구체적인 프롬프트 → 정확한 답변 |

  **AI는 우리가 적어준 "주문서"만큼만 이해하고 대답한다냥!** 지난 시간 Tool의 description이 모호해서 Agent가 헷갈렸던 것도, 사실은 "주문서가 너무 대충 쓰여 있었기 때문"이었던 거다냥! 🐟
</Callout>

## 3단계 · 좋은 프롬프트의 네 가지 핵심 요소 (5~7분)

<NyangSpeech>
  그렇다면 좋은 주문서, 즉 좋은 프롬프트에는 무엇이 들어가야 할까냥?
  딱 **네 가지**만 기억하면 된다냥. 역할 · 맥락 · 지시 · 형식! 😺
</NyangSpeech>

<StepCard step="요소 1" title="역할(Role) — 너는 누구냥?">

AI에게 **어떤 역할을 맡길지** 알려주는 거다냥.

예: `"너는 친절한 여행 가이드야"`, `"너는 10년 차 영양사야"`

역할을 정해주면 AI가 그 사람처럼 말투와 지식을 골라서 대답한다냥!

</StepCard>

<StepCard step="요소 2" title="맥락(Context) — 어떤 상황이냥?">

답변에 필요한 **상황 정보**를 제공하는 거다냥.

예: `"초등학생에게 설명하듯이"`, `"예산은 10만 원이고 2명이서 갈 거야"`

같은 질문이라도 듣는 사람과 상황에 따라 답이 달라지니까 알려주는 거다냥.

</StepCard>

<StepCard step="요소 3" title="지시(Instruction) — 무엇을 어떻게 하냥?">

**무엇을, 어떻게** 해야 하는지 명확하게 짚어주는 거다냥.

예: `"3줄 이내로 요약해줘"`, `"장점과 단점을 각각 알려줘"`

두루뭉술하게 말하지 말고 구체적인 숫자와 조건을 넣자냥!

</StepCard>

<StepCard step="요소 4" title="형식(Format) — 어떤 모양으로 받고 싶냥?">

결과를 **어떤 형태**로 받고 싶은지 지정하는 거다냥.

예: `"표 형식으로"`, `"번호를 매긴 목록으로"`, `"이모지를 붙여서"`

원하는 모양을 미리 말해두면 한 번에 보기 좋은 답이 나온다냥!

</StepCard>

<Callout type="note">
  📌 **기억하기 쉬운 한 문장** — "누가(역할), 어떤 상황에서(맥락), 무엇을 어떻게(지시), 어떤 모양으로(형식)" 답해야 하는지 적어주면 그게 바로 좋은 프롬프트다냥!
</Callout>

### 🐣 기본 코드 미리보기

```python
# 나쁜 프롬프트 (정보가 부족함)
bad_prompt = "여행지 추천해줘"

# 좋은 프롬프트 (역할·맥락·지시·형식이 모두 들어감)
good_prompt = """
너는 친절한 여행 전문가야.
바다를 좋아하는 사람에게,
예산 20만 원으로 갈 수 있는 국내 여행지 3곳을
표 형식으로 추천해줘.
"""
```

같은 `llm`에게 물어봐도, 위 두 프롬프트의 결과는 하늘과 땅 차이다냥! 실습 페이지에서 진짜로 두 결과를 나란히 출력해서 눈으로 확인해볼 거다냥. 🎉

<Callout type="note">
  📌 우리는 **제미나이(Gemini) API**를 쓸 거다냥. OpenAI를 쓰는 자료도 많지만 원리는 똑같다냥. 지난 시간 LangChain Agent에서 쓰던 `ChatGoogleGenerativeAI`를 그대로 재활용한다냥!

  ```python
  # pip install langchain langchain-google-genai  (지난 시간과 동일)
  from langchain_google_genai import ChatGoogleGenerativeAI

  llm = ChatGoogleGenerativeAI(
      model="gemini-3.5-flash",
      google_api_key="여기에 본인의 제미나이 API 키 입력",
  )

  def ask(prompt: str) -> str:
      """제미나이에게 프롬프트를 보내고 답변 텍스트만 받아온다냥"""
      return llm.invoke(prompt).content

  print(ask(good_prompt))
  ```

  앞으로 나오는 예시는 전부 이 `ask()` 함수에 프롬프트만 바꿔 넣으면 된다냥!
</Callout>

## 5단계 · 프롬프트로 할 수 있는 여섯 가지 (7분)

<NyangSpeech>
  좋은 프롬프트 쓰는 법을 배웠으니, 이제 그걸로 **무엇을 할 수 있는지** 보자냥.
  프롬프트 하나만 바꿔도 AI가 여섯 가지나 되는 일을 척척 해낸다냥! 😺
</NyangSpeech>

<StepCard step="활용 1" title="요약(Summarization) — 길면 짧게!">

긴 글을 한 문장으로 줄여달라고 시키는 거다냥.

```python
prompt = """항생제는 세균 감염을 치료하는 약이다냥. 세균을 죽이거나
번식을 막아서 우리 몸의 면역계가 감염과 싸우게 도와준다냥. 다만
바이러스 감염에는 효과가 없고, 잘못 쓰면 항생제 내성이 생긴다냥.

위 내용을 한 문장으로 설명해줘."""
print(ask(prompt))
```

`"초등학생도 알아듣게 설명해줘"` 처럼 **맥락**을 바꾸면 눈높이도 달라진다냥!

</StepCard>

<StepCard step="활용 2" title="질문 답변(Q&A) — 근거 안에서만!">

주어진 **맥락 안에서만** 답하고, 모르면 "잘 모르겠다"고 말하게 시키는 거다냥.

```python
prompt = """아래 맥락을 바탕으로 질문에 짧고 정확하게 답해줘.
답을 확신할 수 없으면 "잘 모르겠다냥"이라고만 답해.

맥락: OKT3는 처음에 생쥐에서 유래한 항체다냥.

질문: OKT3는 처음에 무엇에서 유래했냥?
답변:"""
print(ask(prompt))
```

이렇게 하면 AI가 아무 말이나 지어내는 **환각(hallucination)**을 줄일 수 있다냥!

</StepCard>

<StepCard step="활용 3" title="분류(Classification) — 딱 정해진 답으로!">

글을 정해진 몇 가지 범주 중 하나로 골라내게 시키는 거다냥.

```python
prompt = """다음 문장의 감정을 중립/부정/긍정 중 하나로 분류해줘.

문장: 음식은 그냥 그랬어.
감정:"""
print(ask(prompt))
```

`"그리고 그렇게 분류한 이유도 알려줘"`를 붙이면 **근거**까지 말해준다냥!

</StepCard>

<StepCard step="활용 4" title="역할 놀이(Role Playing) — 캐릭터로 변신!">

AI에게 **역할과 말투**를 정해주고 대화하는 거다냥. (요소 1 '역할' 기억나냥?)

```python
prompt = """너는 과학 연구 조교야. 말투는 전문적이고 과학적이야.
대답은 짧고 간결하게 해줘.

사람: 블랙홀은 어떻게 만들어지냥?
AI:"""
print(ask(prompt))
```

</StepCard>

<StepCard step="활용 5" title="코드 생성(Code) — 프로그램을 대신 짜줘!">

원하는 코드를 자연어로 설명하면 AI가 대신 짜준다냥.

```python
prompt = """테이블 departments(DepartmentId, DepartmentName)
테이블 students(DepartmentId, StudentId, StudentName)
컴퓨터공학과 학생을 모두 조회하는 MySQL 쿼리를 만들어줘."""
print(ask(prompt))
```

</StepCard>

<StepCard step="활용 6" title="추론(Reasoning) — 차근차근 생각하기!">

계산이나 논리 문제는 **단계를 나눠서** 풀라고 시키면 정확해진다냥.

```python
prompt = """이 그룹의 홀수를 모두 더하면 짝수가 될까? 15, 32, 5, 13, 82, 7, 1

단계별로 풀어봐. 먼저 홀수를 찾고, 더한 다음, 결과가 홀수인지 짝수인지 알려줘."""
print(ask(prompt))
```

</StepCard>

## 6단계 · 한 단계 더! 고급 프롬프트 기술 (7분)

<NyangSpeech>
  여기서 조금 더 나아가면 AI를 훨씬 똑똑하게 만들 수 있는 **세 가지 마법**이 있다냥.
  이름은 어렵게 들려도 원리는 아주 간단하니까 겁먹지 말라냥! 🐾
</NyangSpeech>

<StepCard step="기술 1" title="예시 주기(Few-shot) — 몇 개 보여주고 시키기">

지시만 하지 말고, **정답 예시를 몇 개 먼저 보여준 뒤** 새 문제를 시키는 거다냥. AI가 예시를 보고 패턴을 흉내 낸다냥!

```python
prompt = """이 그룹의 홀수를 더하면 짝수야: 4, 8, 9, 15, 12, 2, 1.
A: 답은 거짓이야.

이 그룹의 홀수를 더하면 짝수야: 17, 10, 19, 4, 8, 12, 24.
A: 답은 참이야.

이 그룹의 홀수를 더하면 짝수야: 15, 32, 5, 13, 82, 7, 1.
A:"""
print(ask(prompt))
```

</StepCard>

<StepCard step="기술 2" title="생각의 사슬(CoT) — 풀이 과정을 예시로!">

예시에 정답만 주지 말고 **푸는 과정까지** 보여주는 거다냥. 그러면 AI도 과정을 따라 하며 실수를 줄인다냥!

```python
prompt = """이 그룹의 홀수를 더하면 짝수야: 4, 8, 9, 15, 12, 2, 1.
A: 홀수(9, 15, 1)를 모두 더하면 25야. 답은 거짓이야.

이 그룹의 홀수를 더하면 짝수야: 15, 32, 5, 13, 82, 7, 1.
A:"""
print(ask(prompt))
```

</StepCard>

<StepCard step="기술 3" title="제로샷 CoT — 마법의 한 문장!">

예시를 하나도 안 줘도 되는 가장 간단한 방법이다냥. 그냥 문제 끝에 **"차근차근 생각해보자"** 한 문장만 붙이면 된다냥!

```python
prompt = """사과를 10개 샀어. 이웃에게 2개, 수리 기사에게 2개를 줬어.
그다음 5개를 더 사서 1개를 먹었어. 남은 사과는 몇 개냥?

차근차근 생각해보자."""
print(ask(prompt))
```

이 한 문장만으로도 AI가 곧바로 틀리는 대신 **단계를 밟아** 정답을 찾아낸다냥! ✨

</StepCard>

<Callout type="important">
  ⚠️ 세 기술의 공통점은 **"AI에게 생각할 시간과 본보기를 준다"**는 거다냥. 어려운 문제일수록 곧장 답을 요구하지 말고, 예시를 보여주거나 과정을 밟게 하면 훨씬 똑똑해진다냥!
</Callout>

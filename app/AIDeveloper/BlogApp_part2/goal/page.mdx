## 2단계 · 이미지 생성 프롬프트 설계하기 (5~7분)

<NyangSpeech>
  11차시에서 배운 거 기억나냥? 원하는 이미지를 얻으려면 **구체적으로 묘사**해야 한다냥!
  "고양이"라고만 하면 AI 화가는 막막하지만, "햇살 드는 창가에서 낮잠 자는 노란 고양이,

</NyangSpeech>

### 핵심 비유 — "글에 어울리는 표지 그리기"

잡지나 블로그 글에는 내용을 한눈에 보여주는 **대표 사진(표지)**이 있지냥. 오늘 우리가 하는 일은, 우리가 쓴 글의 분위기를 **AI 화가에게 설명해서**, 그 글에 딱 어울리는 표지 그림을 그려달라고 부탁하는 거다냥.

<Callout type="analogy">
  **글(작가) + 그림(화가) = 한 팀!**

  - **작가**는 주제를 받아 글을 쓴다냥.
  - **화가**는 그 글의 분위기를 듣고 어울리는 표지를 그린다냥.

  둘이 한 팀이 되어 **완성된 한 편의 블로그 글**을 만드는 거다냥!
</Callout>

### 이미지 프롬프트도 결국 "좋은 프롬프트"

<StepCard step="핵심" title="좋은 이미지 프롬프트의 원리는 똑같다냥!">

이미지 프롬프트라고 특별한 게 아니다냥. 우리가 글 프롬프트에서 배운 원리가 그대로 적용된다냥.

| 원리 | 예시 |
|---|---|
| **구체적 묘사** | "카페"보다 "원목 테이블과 작은 화분이 있는 아늑한 홈카페" |
| **분위기** | "따뜻한", "차분한", "활기찬" 같은 느낌 단어 |
| **스타일** | "수채화 느낌", "사진처럼 사실적인", "귀여운 일러스트" |

게다가 **글의 주제에서 이미지 키워드를 뽑아내는 과정** 자체를 AI에게 시킬 수도 있다냥! 즉, AI에게 "이 글의 대표 이미지를 묘사하는 영어 문장을 만들어줘"라고 부탁하는 거다냥.

</StepCard>

### 기본 코드 예시 — 주제에서 이미지 묘사 뽑아내기

```python
from google import genai

client = genai.Client(api_key="여기에 본인의 제미나이 API 키 입력")

topic = "초보자를 위한 홈카페 만들기"

image_desc_prompt = f"""
'{topic}'이라는 블로그 글의 대표 이미지를 만들려고 해.
이미지 생성 AI에게 전달할 영어 묘사문을 한 문장으로 만들어줘.
따뜻하고 아늑한 분위기를 담아줘.
"""
image_description = client.models.generate_content(model="gemini-3.5-flash", contents=image_desc_prompt).text
print("이미지 묘사:", image_description)
```

이렇게 하면 AI가 주제를 보고 **"warm and cozy home cafe with wooden table and small plants..."** 같은 영어 묘사문을 만들어준다냥. 그 묘사문을 이미지 생성 AI에게 그대로 전달하면 표지 그림이 뿅! 하고 나온다냥.

### 핵심 비유 — "글과 그림을 함께 내놓는 출판사"

<Callout type="analogy">
  오늘 만들 앱은 작은 **출판사**와 같다냥.

  - **주제(원고 의뢰)**가 들어오면 →
  - **작가(글 생성 AI)**가 글을 쓰고 →
  - **화가(이미지 생성 AI)**가 표지를 그려서 →
  - 한 편의 완성된 글을 **독자(사용자)**에게 내놓는 거다냥.

  여러 AI 기능을 **한 화면에서 협업**하게 만드는 게 오늘의 핵심이다냥!
</Callout>

<Callout type="note">
  오늘 만든 앱에서 AI는 한 가지 일만 하는 게 아니라냥. **글 구조 만들기, 본문 쓰기,
  이미지 묘사 만들기**처럼 여러 일을 동시에 해낸다냥. 이렇게 여러 AI 기능을 묶어
  하나의 앱으로 만드는 걸 **멀티 기능 통합**이라고 부른다냥!
</Callout>

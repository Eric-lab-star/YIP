## ★ 핵심 비유 · "재료 손질 담당 + 요리 담당이 한 팀이 되다" 🍳

<Callout type="analogy">
  - `get_weather` 함수는 날씨 재료를 손질해서 가져오는 **재료 담당**이다냥.
  - 제미나이(LLM)는 그 재료로 요리를 만드는 **요리 담당**이다냥.
  - LangChain의 Agent는 이 둘을 연결해주는 **주방장**이다냥.

  주문(질문)을 들으면 주방장이 *"이건 재료 담당한테 먼저 부탁해야겠다"*고 판단하고, 받은 재료로 요리 담당이 최종 요리(답변)를 완성하게 시키는 거다냥! 🐾
</Callout>

<MissionCard number={2} title="날씨 함수를 Tool로 만들기 🔧" time="8~10분">

지난 시간에 만든 **날씨 API 호출 함수**를 다듬고, LangChain의 `@tool`로 감싸볼 거다냥. 아래 템플릿에서 **빈칸(`____`, `___`)을 직접 채워보자냥.**

```python
from langchain.agents import create_agent
from langchain_core.tools import tool
from langchain_google_genai import ChatGoogleGenerativeAI
import requests

# 지난 시간 만든 날씨 API 함수에 @tool을 붙여 LangChain 도구로 등록
# 함수 설명문(docstring)이 곧 도구 설명이 된다냥!
@tool
def get_weather(city: str) -> str:
    """도시 이름을 입력받아 현재 날씨 정보를 알려줍니다."""
    weather_api_key = "여기에 본인의 날씨 API 키 입력"
    url = f"https://api.openweathermap.org/data/2.5/weather?q={city}&appid={____}&units=metric"

    response = requests.get(url)
    data = response.json()

    temperature = data["main"]["___"]
    description = data["weather"][0]["description"]

    return f"{city}의 현재 온도는 {temperature}도이고, 날씨는 {description}입니다."
```

> ✅ **확인해보자냥 (빈칸 힌트)**
> - `{____}` 자리에는 어떤 변수가 들어가야 할까냥? (API 키를 담아둔 변수 이름이다냥!)
> - `data["main"]["___"]` 에서 온도 값을 가져오는 키는 무엇이냥? (영어로 "온도"라냥 🌡️)

<Callout type="tip">
  💡 `@tool` 함수의 **설명문(docstring)**이 핵심이다냥! Agent는 이 설명을 읽고 도구를 쓸지 판단한다냥. 설명이 모호하면 Agent가 도구를 못 찾으니, **구체적으로** 써주자냥.
</Callout>

</MissionCard>

<MissionCard number={3} title="Agent 만들고 실행하기 🎯" time="8~10분">

이제 제미나이 **LLM**과 방금 만든 **Tool**을 연결해서 **Agent**를 만들고, 질문을 던져볼 거다냥!

```python
# 3) 제미나이 LLM 준비
llm = ChatGoogleGenerativeAI(
    model="gemini-3.5-flash",
    google_api_key="여기에 본인의 제미나이 API 키 입력"
)

# 4) Agent 만들기 (LLM + 도구를 한 팀으로 묶는다냥)
agent = create_agent(llm, [get_weather])

# 5) 질문 던지기 - 직접 다양한 질문을 작성해보세요
question = ____
result = agent.invoke({"messages": [{"role": "user", "content": question}]})

# Agent가 도구를 어떻게 썼는지 전체 과정을 들여다보자냥!
for message in result["messages"]:
    message.pretty_print()
```

> ✅ **확인해보자냥 (빈칸 힌트)**
> - `question = ____` 자리에 `"오늘 서울 날씨 알려줘"` 같은 문장을 넣어보자냥.
> - `{"messages": [{"role": "user", "content": question}]}` 모양으로 질문을 넣는다냥. 답은 `result["messages"][-1].content` 에 담겨 있다냥.
> - 실행하면 Agent가 스스로 `get_weather` 도구를 호출해서 답하냥?

<Callout type="tip">
  🐾 **`message.pretty_print()` 가 마법이다냥!** 결과의 `messages`를 하나씩 출력하면 Agent가 *"생각하는 과정"*이 화면에 주르륵 나온다냥. "이 질문엔 날씨 도구가 필요하겠군 → 도구 호출 → 결과 받음 → 최종 답 작성" 하는 흐름을 직접 눈으로 읽어보자냥. 이해에 정말 큰 도움이 된다냥!
</Callout>

</MissionCard>

<MissionCard number={4} title="다양한 질문으로 테스트하기 🔬" time="5~7분">

이번엔 여러 질문을 던져보면서 **"AI가 스스로 판단한다"**는 걸 직접 체감해보자냥!

- 🌤️ **날씨 관련 질문**: `"부산 날씨 알려줘"` → Agent가 날씨 도구를 **사용**하는지 확인하자냥.
- 🍱 **날씨와 무관한 질문**: `"오늘 점심 메뉴 추천해줘"` → Agent가 도구를 **쓰지 않고** 바로 답하는지 확인하자냥.

```python
# 날씨 도구를 쓰는 질문
result = agent.invoke({"messages": [{"role": "user", "content": "부산 날씨 알려줘"}]})
print(result["messages"][-1].content)

# 날씨 도구가 필요 없는 질문
result = agent.invoke({"messages": [{"role": "user", "content": "오늘 점심 메뉴 추천해줘"}]})
print(result["messages"][-1].content)
```

> ✅ **확인해보자냥**
> - 두 질문에서 `messages`(대화 과정)가 어떻게 다르냥? (`pretty_print()`로 펼쳐보자냥)
> - 날씨 질문일 때만 도구를 호출했냥? 그게 바로 **Agent가 스스로 판단한 증거**다냥! 🎉

<Callout type="important">
  ⚠️ **만약 날씨 질문인데도 도구를 안 쓴다면?** `@tool` 함수의 설명문(docstring)이 너무 모호한 경우가 많다냥. "현재 날씨, 온도를 도시별로 알려준다"처럼 더 **구체적으로** 고쳐보자냥.
</Callout>

</MissionCard>

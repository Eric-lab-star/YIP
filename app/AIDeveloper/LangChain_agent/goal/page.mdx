## 1단계 · LangChain이 왜 필요하냥? (5분)

<NyangSpeech>
  지난 시간을 떠올려보자냥. 우리는 날씨 API를 호출해서 "서울은 23도, 맑음" 같은
  데이터를 받았지? 그리고 그 값을 **우리 손으로** 프롬프트에 적어서 제미나이에게
  줬다냥. 제미나이는 우리가 적어준 메모만 보고 답을 했던 거다냥.
</NyangSpeech>

지난 시간 방식의 흐름을 정리하면 이렇다냥.

1. **사람**이 날씨 API를 호출한다냥.
2. **사람**이 받아온 데이터를 프롬프트 문자열에 끼워 넣는다냥.
3. 제미나이는 그 프롬프트만 읽고 답한다냥.

여기서 모든 "판단"과 "준비"는 **사람**이 했다냥. AI는 시키는 대로 답만 한 거지냥. 그런데 LangChain을 쓰면, 이 판단과 도구 사용을 **AI에게 맡길 수** 있다냥!

## 2단계 · 핵심 개념 세 가지 (5~7분)

<NyangSpeech>
  LangChain을 이해하려면 딱 세 단어만 알면 된다냥. **LLM, Tool, Agent**!
  이 셋이 한 팀이 되어서 똑똑하게 움직이는 거다냥. 하나씩 알려줄게냥.
</NyangSpeech>

<StepCard step="개념" title="LangChain의 3대 주역이다냥!">

| 이름 | 역할 | 한마디로 |
|---|---|---|
| **LLM (제미나이)** | 생각하고 답을 만드는 두뇌 | 똑똑한 머리 |
| **Tool (도구)** | 날씨 API처럼 LLM이 필요할 때 "사용"하는 외부 기능 | 손에 쥔 도구 |
| **Agent (에이전트)** | 어떤 도구를 쓸지 스스로 판단하고, 결과로 최종 답을 만드는 역할 | 결정하는 지휘자 |

</StepCard>

여기서 가장 중요한 건 **Agent**다냥. Agent는 사용자의 질문을 보고, "이 질문에 답하려면 날씨 도구를 써야겠는걸?"이라고 **스스로 판단**한다냥. 그리고 도구를 직접 호출해서 결과를 받아온 뒤, 그걸 바탕으로 최종 답을 만들어준다냥!

## 3단계 · 코드로 살짝 맛보기 (개념 미리보기)

<NyangSpeech>
  자세한 작성은 실습 페이지에서 직접 해볼 거다냥. 여기서는 "아, LangChain 코드는
  대충 이렇게 생겼구나" 정도로 큰 그림만 눈에 익혀보자냥!
</NyangSpeech>

LangChain으로 날씨 Agent를 만드는 큰 흐름은 이렇다냥.

```python
from langchain.agents import create_agent
from langchain_core.tools import tool
from langchain_google_genai import ChatGoogleGenerativeAI

# 1) 날씨 함수를 @tool로 감싸 도구로 등록 (도구를 만든다냥)
@tool
def get_weather(city: str) -> str:
    """도시 이름을 입력받아 현재 날씨 정보를 알려준다."""
    ...  # 실제 날씨 API 호출

# 2) 제미나이 LLM 준비 (두뇌를 준비한다냥)
llm = ChatGoogleGenerativeAI(model="gemini-3.5-flash", google_api_key="...")

# 3) Agent 만들기 (두뇌 + 도구를 한 팀으로 묶는다냥)
agent = create_agent(llm, [get_weather])

# 4) 질문 던지기 → Agent가 스스로 판단해서 답한다냥
result = agent.invoke({"messages": [{"role": "user", "content": "부산 날씨 알려줘"}]})
print(result["messages"][-1].content)
```

보면, 우리가 하는 일은 **도구를 등록하고(@tool) → 두뇌를 준비하고(LLM) → 둘을 묶기(Agent)**까지다냥. 그다음 질문을 던지면, 도구를 쓸지 말지는 **Agent가 알아서** 결정한다냥!

<Callout type="tip">
  `@tool` 함수의 **설명문(docstring)**이 정말 중요하다냥! Agent는 이 설명을 읽고 "아, 이 도구가 날씨를 알려주는구나"라고 판단한다냥. 그래서 도구 설명은 **구체적으로** 써줘야 Agent가 잘 찾아 쓴다냥!
</Callout>

<MissionCard number={2} title="Chroma DB 만들고 데이터 저장하기 " time="8~10분">

이제 서랍장(컬렉션)을 만들고, 청크들을 저장해볼 거다냥. 개념 시간에 본 것처럼, 한국어 검색이 잘 되도록 **제미나이 임베딩을 연결**해서 만들 거다냥. `GeminiEmbedding` 클래스는 그대로 복사해서 쓰면 된다냥! 아래 템플릿에서 **빈칸(`____`)을 직접 채워보자냥.**

```python
import chromadb
from chromadb import Documents, EmbeddingFunction, Embeddings
from google import genai

genai_client = genai.Client(api_key="여기에 본인의 제미나이 API 키 입력")

# Chroma에게 "임베딩은 제미나이에게 맡겨라"라고 알려주는 연결 코드다냥 (그대로 복사!)
class GeminiEmbedding(EmbeddingFunction):
    def __call__(self, input: Documents) -> Embeddings:
        result = genai_client.models.embed_content(
            model="gemini-embedding-001",
            contents=list(input),
        )
        return [e.values for e in result.embeddings]

# Chroma 클라이언트 생성 (메모리에 임시로 저장)
client = chromadb.Client()

# 컬렉션(서랍장) 만들기 — 제미나이 임베딩을 연결한다냥
collection = client.create_collection(
    name="my_documents",
    embedding_function=GeminiEmbedding(),
)

# 지난 시간 만든 chunks 리스트를 그대로 사용 (예시로 재정의)
chunks = [
    "여기에 문서 청크 1 내용",
    "여기에 문서 청크 2 내용",
    "여기에 문서 청크 3 내용",
]

# 각 청크에 고유한 id를 부여해 저장해보세요
ids = [____ for i in range(len(chunks))]  # 예: "chunk_0", "chunk_1", ...

collection.add(
    documents=chunks,
    ids=ids
)

print(f"{collection.count()}개의 청크가 저장되었습니다.")
```

> **확인해보자냥 (빈칸 힌트)**
> - `____` 자리에는 청크마다 **다른 id**를 만들어주는 식이 들어가야 한다냥.
>   `i`를 활용해서 `f"chunk_{i}"` 처럼 만들면 `chunk_0`, `chunk_1`, ... 이 된다냥!
> - 마지막 줄에 저장된 청크 개수가 잘 출력되냥?

<Callout type="important">
  **id는 절대 중복되면 안 된다냥!** 같은 id로 또 `add` 하면 에러가 날 수 있으니,
  꼭 청크마다 서로 다른 id를 붙여주자냥.
</Callout>

</MissionCard>

<MissionCard number={3} title="DB에서 검색하기 " time="8~10분">

이번엔 저장한 서랍장에 **질문을 던져서** 가장 비슷한 청크를 꺼내볼 거다냥! `query_texts`에 넣을 **질문을 직접 작성**해보자냥.

```python
# 질문을 직접 작성해보세요
question = ____

results = collection.query(
    query_texts=[question],
    n_results=2  # 가장 유사한 상위 2개 결과
)

print("검색된 내용:")
for doc in results["documents"][0]:
    print("-", doc)
```

> **확인해보자냥 (빈칸 힌트)**
> - `question = ____` 자리에는 검색하고 싶은 문장을 **따옴표로 감싼 문자열**로 넣어야 한다냥.
>   예: `question = "고양이에 대해 알려줘"`
> - 질문과 가장 비슷한 청크 2개가 잘 출력되냥?

<Callout type="tip">
  우리가 직접 임베딩을 만들거나 유사도를 계산하지 않았는데도 검색이 됐지?
  그게 바로 벡터DB의 마법이다냥! `query()`를 부르면 Chroma가 질문을 (우리가
  연결해준 제미나이 임베딩으로) 변환한 다음, **알아서 가장 가까운 자리에 있는
  문서**를 찾아서 건네준다냥.
</Callout>

</MissionCard>

<MissionCard number={4} title="새 문서 추가 + 재검색하기 " time="5~7분">

서랍장에 **새 책을 추가**하면 검색 결과가 어떻게 달라질까냥? 새로운 청크를 직접 만들어 추가하고, **같은 질문**으로 다시 검색해서 결과를 비교해보자냥!

```python
# 새로운 청크(문장)를 1~2개 직접 작성해보세요
new_chunks = [____, ____]
new_ids = ["chunk_new_1", "chunk_new_2"]

collection.add(
    documents=new_chunks,
    ids=new_ids
)

print(f"이제 총 {collection.count()}개의 청크가 저장되어 있습니다.")

# 같은 질문으로 다시 검색해서 결과가 달라지는지 확인해보세요
results = collection.query(
    query_texts=[question],
    n_results=2
)

print("재검색 결과:")
for doc in results["documents"][0]:
    print("-", doc)
```

> **확인해보자냥 (빈칸 힌트)**
> - `new_chunks` 의 `____` 두 자리에는 새로 추가할 **문장(문자열)**을 넣으면 된다냥.
>   예: `new_chunks = ["고양이는 생선을 좋아한다", "고양이는 낮잠을 즐긴다"]`
> - 새 문서를 넣었더니 **재검색 결과가 달라졌냥?** 새 청크가 질문과 더 비슷하다면
>   결과 위로 올라올 수도 있다냥!

**직접 해보기**
시간이 남으면 `n_results` 값을 `2` → `3`, `4`로 바꿔가며 검색해보자냥. 꺼내오는 문서 개수가 어떻게 달라지냥?

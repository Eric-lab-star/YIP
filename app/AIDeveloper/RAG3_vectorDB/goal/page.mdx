### 임시 메모지 vs 정리된 서랍장

지난 시간 우리가 만든 검색기는 **매번 책상 위에 책을 전부 펼쳐놓고**, 하나하나 다시 분류한 다음 찾는 것과 같았다냥. 책이 많아질수록 책상은 난장판이 되고, 찾는 시간도 점점 길어지겠지?

벡터DB는 **이미 분류가 끝난 서랍장**이다냥. 한 번 책을 분류해서 서랍에 쏙 넣어두면, 다음에 찾을 때는 **서랍만 열어서 바로 꺼내면** 된다냥. 게다가 새 책이 들어와도 그 책 한 권만 분류해서 서랍에 추가하면 되니까 훨씬 효율적이라냥!

<Callout type="analogy">
  **비유로 이해하기 — 임시 메모지 vs 정리된 서랍장**

  | 지난 시간 검색기 | 오늘의 벡터DB(Chroma) |
  |---|---|
  | 매번 책을 다 펼쳐 다시 분류 | 한 번 분류해 서랍에 저장 |
  | 찾을 때마다 임베딩을 새로 계산 | 저장된 임베딩을 그대로 재사용 |
  | 책이 많아지면 점점 느려짐 | 서랍만 열면 바로 꺼냄 |
  | 새 책이 와도 전체를 다시 분류 | 새 책 한 권만 서랍에 추가 |

  한 번 정리해두면 두고두고 빠르게 꺼내 쓸 수 있다냥!
</Callout>

## 3단계 · Chroma 기본 사용법 (7~10분)

<NyangSpeech>
  이제 Chroma를 어떻게 쓰는지 알려줄 거다냥. 딱 세 가지 동작만 기억하면
  된다냥 — **창고 열기 → 문서 넣기(add) → 질문으로 꺼내기(query)**!
  생각보다 진짜 간단하다냥
</NyangSpeech>

### 자동으로 분류해주는 마법의 서랍장

Chroma에 `collection.add()`로 문서를 넣으면, 서랍장이 **알아서 그 문서를 임베딩으로 변환해** 적절한 자리에 정리해준다냥. 그리고 `collection.query()`로 질문을 하면, 서랍장이 **가장 가까운 자리에 있는 문서들을 꺼내서** 건네준다냥.

지난 시간 우리가 손으로 직접 했던 임베딩 변환과 유사도 계산을, 이제는 **서랍장이 자동으로** 해주는 거라냥!

<StepCard step="핵심" title="Chroma, 이 세 가지만 기억하면 된다냥!">

| 동작 | 코드 | 하는 일 |
|---|---|---|
| 컬렉션 만들기 | `client.create_collection()` | 문서를 담을 **서랍장(창고)** 만들기 |
| 저장하기 | `collection.add()` | 문서를 넣으면 **자동으로 임베딩**해서 저장 |
| 검색하기 | `collection.query()` | 질문에 **가장 비슷한 문서**를 꺼내주기 |

</StepCard>

### 기본 코드 예시 — 서랍장 만들고 문서 넣기

```python
import chromadb

# Chroma 클라이언트 생성 (메모리에 임시로 저장)
client = chromadb.Client()

# 컬렉션(서랍장) 만들기
collection = client.create_collection(name="my_documents")

# 문서를 id와 함께 저장
collection.add(
    documents=["고양이는 귀엽다", "강아지는 충성스럽다"],
    ids=["doc_0", "doc_1"]
)

print(f"{collection.count()}개의 문서가 저장되었습니다.")
```

### 기본 코드 예시 — 질문으로 꺼내기

```python
# 질문과 가장 비슷한 문서를 검색
results = collection.query(
    query_texts=["귀여운 동물은?"],
    n_results=1
)

print(results["documents"][0])
```

<Callout type="tip">
  **꼭 기억할 두 가지**

  - `collection.add()`를 호출하면 **임베딩이 자동으로 생성**된다냥. 지난 시간 우리가
    손으로 했던 그 변환을 Chroma가 알아서 해준다냥!
  - **id는 절대 중복되면 안 된다냥.** 같은 id로 다시 add하면 에러가 날 수 있으니
    `chunk_0`, `chunk_1`처럼 서로 다른 이름을 붙여주자냥.
</Callout>

<Callout type="note">
  "어? 지난 시간 제미나이 임베딩이랑 다른 거 아니냥?" 맞다냥! Chroma는
  내부적으로 **자기만의 임베딩 방식**을 자동으로 사용한다냥. 오늘은 벡터DB가
  어떻게 저장하고 검색하는지 그 **동작 방식을 체험**하는 게 목표라서, 기본
  임베딩으로도 충분하다냥
</Callout>

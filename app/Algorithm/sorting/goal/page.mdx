## 2단계 · sorted 와 sort 는 다르다냥 (7분)

```python
arr = [3, 1, 2]

new = sorted(arr)     # 새 리스트를 만들어 돌려준다냥
print(new)            # [1, 2, 3]
print(arr)            # [3, 1, 2]   원본 그대로!

arr.sort()            # 원본을 직접 바꾼다냥
print(arr)            # [1, 2, 3]
```

<Callout type="important">
  `arr.sort()` 는 **아무것도 돌려주지 않는다냥.**

  ```python
  arr = sorted(arr)     # 맞다냥
  arr = arr.sort()      # 틀렸다냥! arr 이 None 이 된다냥
  ```

  두 번째 줄은 에러도 안 나고 조용히 `None` 이 들어간다냥. 그 다음 줄에서
  이상한 에러가 터져서 원인을 찾기 어렵다냥. 헷갈리면 **`sorted()` 만** 쓰면
  된다냥.
</Callout>

<StepCard step="정리" title="언제 뭘 쓰냥">
  - **원본을 남겨야 한다** → `sorted(arr)`
  - **원본을 그냥 바꿔도 된다** → `arr.sort()`

  잘 모르겠으면 `sorted()` 를 쓰자냥. 원본이 안 바뀌니 사고가 안 난다냥.
</StepCard>

## 3단계 · 거꾸로 세우기 (3분)

```python
arr = [3, 1, 4, 1, 5]

print(sorted(arr))                  # [1, 1, 3, 4, 5]
print(sorted(arr, reverse=True))    # [5, 4, 3, 1, 1]
print(sorted(arr)[::-1])            # [5, 4, 3, 1, 1]  같은 결과라냥
```

<Callout type="tip">
  **상위 k개**를 뽑는 건 이 둘을 붙이면 끝이라냥.

  ```python
  top3 = sorted(arr, reverse=True)[:3]
  ```

  큰 것부터 줄을 세우고, 앞에서 세 개를 자른다냥. 지난 시간에 배운 슬라이싱이
  여기서 바로 쓰인다냥.
</Callout>

## 4단계 · 기준을 바꾸는 key (10분)

<NyangSpeech>
  오늘의 핵심이라냥. 지금까지는 **값 자체**로 줄을 세웠는데, `key=` 를 주면
  **다른 기준**으로 세울 수 있다냥.
</NyangSpeech>

```python
words = ["banana", "kiwi", "fig"]

print(sorted(words))                  # ['banana', 'fig', 'kiwi']   사전순
print(sorted(words, key=len))         # ['fig', 'kiwi', 'banana']   길이순
```

<StepCard step="원리" title="key 는 '무엇을 보고 비교할지'">
  `key=len` 은 이런 뜻이라냥 — **"각 단어를 `len()` 에 넣어본 결과로 비교해라."**

  | 단어 | `len(단어)` | 이 값으로 줄 세움 |
  |---|---|---|
  | banana | 6 | 3등 |
  | kiwi | 4 | 2등 |
  | fig | 3 | 1등 |

  단어 자체가 아니라 **길이라는 잣대**로 비교한 거라냥.
</StepCard>

### 내 마음대로 기준 만들기 — lambda

`len` 같은 이름 있는 함수 말고, 그 자리에서 기준을 만들 수도 있다냥.

```python
arr = [-5, 2, -1, 4]

print(sorted(arr, key=abs))              # [-1, 2, 4, -5]   절댓값 기준
print(sorted(arr, key=lambda x: -x))     # [4, 2, -1, -5]   부호를 뒤집어 큰 순
```

<Callout type="tip">
  `lambda x: -x` 는 "x를 받아서 -x를 돌려주는 이름 없는 함수"라냥. 지금은
  **`key=` 자리에 쓰는 짧은 규칙**이라고만 알아둬도 충분하다냥.
</Callout>

## 5단계 · 1순위, 2순위 — 튜플 키 (10분)

<NyangSpeech>
  실전에서 제일 많이 쓰는 기술이라냥. **"길이순으로, 길이가 같으면 사전순"**
  같은 조건이라냥.
</NyangSpeech>

```python
words = ["dd", "a", "ccc", "bb", "ab"]

print(sorted(words, key=lambda s: (len(s), s)))
# ['a', 'ab', 'bb', 'dd', 'ccc']
```

<StepCard step="원리" title="튜플은 앞에서부터 비교한다냥">
  `(len(s), s)` 는 **비교할 잣대를 순서대로 늘어놓은 것**이라냥.

  1. 먼저 `len(s)` 로 비교한다냥
  2. 그게 **같으면** 그때 `s` 로 비교한다냥

  | 단어 | 키 | 순서 |
  |---|---|---|
  | a | (1, "a") | 1등 |
  | ab | (2, "ab") | 2등 |
  | bb | (2, "bb") | 3등 |
  | dd | (2, "dd") | 4등 |
  | ccc | (3, "ccc") | 5등 |

  길이 2인 셋은 `ab`, `bb`, `dd` 순으로 사전순 정렬됐다냥. 3순위, 4순위가
  필요하면 튜플에 계속 이어붙이면 된다냥.
</StepCard>

## 6단계 · 정렬 뒤에 조심할 것 (7분)

<NyangSpeech>
  정렬만 하면 끝이 아니라냥. 여기서 틀리는 경우가 많다냥.
</NyangSpeech>

### 함정 하나 — 중복

**두 번째로 큰 수**를 구한다고 하자냥.

```python
arr = [5, 5, 3]

print(sorted(arr)[-2])    # 5   ...이게 맞냥?
```

<Callout type="important">
  `[5, 5, 3]` 을 정렬하면 `[3, 5, 5]` 라서 뒤에서 두 번째는 **5** 라냥.

  그런데 "서로 다른 값 중 두 번째로 큰 값"을 원했다면 답은 **3** 이라냥!

  중복을 하나로 세려면 먼저 **`set()` 으로 중복을 없애야** 한다냥.

  ```python
  print(sorted(set(arr))[-2])    # 3
  ```

  문제를 읽을 때 **중복을 어떻게 세라고 했는지** 꼭 확인하자냥. 이걸 놓쳐서
  틀리는 경우가 정말 많다냥.
</Callout>

### 함정 둘 — 가운데 인덱스

`n` 개를 정렬했을 때 가운데는 몇 번이냥?

```python
arr = sorted([3, 1, 4, 1, 5])    # [1, 1, 3, 4, 5]
n = len(arr)
print(arr[n // 2])               # 3
```

<StepCard step="확인" title="왜 n // 2 냥">
  `n` 이 5면 인덱스는 0, 1, **2**, 3, 4 라냥. 가운데는 2번이고 `5 // 2` 가
  2라서 딱 맞는다냥.

  `n // 2 - 1` 이나 `n / 2` 로 쓰면 틀린다냥. `/` 는 소수를 만들어서 인덱스로
  못 쓴다냥. **정수 나눗셈 `//`** 를 써야 한다냥.
</StepCard>

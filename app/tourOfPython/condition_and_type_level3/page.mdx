# 조건문과 타입 Level 3

## 문제 1. 은행 ATM 프로그램

### 아래 조건을 모두 만족하는 ATM 프로그램을 작성하세요.

```text
초기 잔액: 10000원

조건 1. 입력값이 숫자가 아니면 → "숫자만 입력해 주세요."
조건 2. 입력한 금액이 0 이하이면 → "1원 이상 입력해 주세요."
조건 3. 입력한 금액이 잔액보다 많으면 → "잔액이 부족합니다. 현재 잔액: {잔액}원"
조건 4. 출금 성공 시 → "출금 완료. 현재 잔액: {잔액}원"

실행 예시 1)
출금할 금액을 입력하세요: abc
숫자만 입력해 주세요.

실행 예시 2)
출금할 금액을 입력하세요: -500
1원 이상 입력해 주세요.

실행 예시 3)
출금할 금액을 입력하세요: 50000
잔액이 부족합니다. 현재 잔액: 10000원

실행 예시 4)
출금할 금액을 입력하세요: 3000
출금 완료. 현재 잔액: 7000원
```

## 문제 2. 나누기 계산기

### 두 숫자를 입력받아 나눗셈 결과를 출력하는 프로그램을 작성하세요.

```text
조건 1. 입력값이 숫자가 아니면 → "숫자만 입력해 주세요."
조건 2. 나누는 수가 0이면 → "0으로 나눌 수 없습니다."
조건 3. 결과가 양수이면 → "결과: {값} (양수)"
조건 4. 결과가 음수이면 → "결과: {값} (음수)"
조건 5. 결과가 0이면 → "결과: 0"

힌트: 0으로 나눌 때 발생하는 ZeroDivisionError도 except로 잡아야 해요.

실행 예시 1)
첫 번째 숫자: 10
두 번째 숫자: abc
숫자만 입력해 주세요.

실행 예시 2)
첫 번째 숫자: 10
두 번째 숫자: 0
0으로 나눌 수 없습니다.

실행 예시 3)
첫 번째 숫자: 10
두 번째 숫자: 4
결과: 2.5 (양수)
```

## 문제 3. 온라인 쇼핑몰 주문 시스템

### 상품 번호, 수량, 포인트 사용 여부를 입력받아 최종 결제 금액을 출력하는 프로그램을 작성하세요.

```text
상품 목록
1번 노트북   1,200,000원
2번 마우스      25,000원
3번 키보드      55,000원

조건 1. 입력값이 숫자가 아니면 → "숫자만 입력해 주세요."
조건 2. 없는 상품 번호이면 → "존재하지 않는 상품입니다."
조건 3. 수량이 1 미만이면 → "수량은 1개 이상이어야 합니다."
조건 4. 보유 포인트: 5,000점 (1점 = 1원)
        포인트 사용을 원하면 → 총 금액에서 5,000원 차감
        단, 총 금액이 포인트보다 적으면 → "포인트가 결제 금액보다 많습니다."
조건 5. 최종 금액이 100,000원 이상이면 → 10% 할인 추가 적용

실행 예시 1)
상품 번호를 입력하세요: 두번
숫자만 입력해 주세요.

실행 예시 2)
상품 번호를 입력하세요: 2
수량을 입력하세요: 3
포인트를 사용하시겠습니까? (y/n): y
총 금액: 75,000원
포인트 적용: -5,000원
최종 결제 금액: 70,000원

실행 예시 3)
상품 번호를 입력하세요: 2
수량을 입력하세요: 10
포인트를 사용하시겠습니까? (y/n): y
총 금액: 250,000원
포인트 적용: -5,000원
10% 할인 적용: -24,500원
최종 결제 금액: 220,500원
```

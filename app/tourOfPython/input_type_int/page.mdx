# input() type() int()

<span id="input" />

## input()

```python
age = input("몇 살이에요")
phonenumber = input("전화번호 뭐에요")
```

`input` 함수를 이용해서 사용자에게 질문을 할 수 있어요. 그리고 사용자가 답을 할 때까지 기다리고 답변을 변수에 저장시킬 수 있어요.

`input` 함수는 `print` 함수와 마찬가지로 `built-in` 함수에요. `built-in` 함수는 나중에 더 자세하게 이야기하겠지만, 지금은 미리 만들어진 파이썬 프로그램에 항상 들어가는 함수라고 생각하세요.

<span id="type" />

## type()

![파이썬의 자료형](https://r2.kimkyungsub.com/types.jpg)

type 함수는 값의 형태를 알려주는 기능을 해요. 그렇다면 코드에서 의미하는 형태란 무엇일까요? 형태란 마치 동그라미, 세모, 네모 같은 모양이라고 생각하면 돼요! 동그라미 구멍에는 동그라미만 넣을 수 있고, 세모 구멍에는 세모만 넣을 수 있어요. 코드도 마찬가지에요. 타입이 같지 않으면 오류가 생겨요. 그렇기 때문에 타입이 같은지 확인하는 기능을 type 함수를 이용해서 하는 거예요.

### 연습하기

아래의 코드를 작성하고 나오는 결과를 확인하세요.

```python
age = type(20)  # type에 20을 전달
print(age)      # 20의 타입을 알려준다.
```

`type`이라는 함수도 `built-in` 함수예요. 확인하고 싶은 값을 type 함수의 인수로 주면 type 함수로부터 타입을 확인할 수 있어요.

이 코드를 실행했을 때 어떤 결과가 나왔나요? `<class 'int'>` 이런 알 수 없는 영어 단어들이 나와도 겁먹지 않아도 돼요. `class`는 나중에 천천히 알아보도록 하고 지금은 그 다음에 오는 단어만 보면 돼요. 우리가 일상생활에서 주로 사용하는 숫자는 모두 `int`라고 생각해도 좋아요.

### 연습하기

아래의 코드를 작성하고 나오는 결과를 확인하세요.

```python
age = input("age?: ")
print(type(age))
```

이번에는 `input()`으로 받은 값의 타입을 확인해볼까요? 분명 숫자를 입력했는데 `int`라고 안 나오고, `str`이라고 나와요. `str`은 문자열이라는 의미에요. `input`은 항상 문자열 값만 반환해요. 그렇다면 `int`를 받고 싶을 때는 어떻게 해야 할까요.

<span id="int" />

## int()

```python
age = int(input("age? :"))
print(type(age))
```

`int` 값으로 바꾸고 싶을 때는 `int()`를 사용해요. 이 함수를 사용하면 "23"과 같은 문자열을 입력받았을 때 23과 같은 숫자로 바꾸어주는 기능을 해요.

```python
ValueError: invalid literal for int() with base 10: 'asfasfa'
```

하지만 숫자로 바꿀 수 없는 문자열도 있어요. 예를 들면 사용자가 "hello"와 같은 글자를 입력했을 경우에요. 이런 경우에 `int()`를 실행하면 어떤 결과가 나오는지 확인해봐요. 아마 ValueError라고 시작되는 문장을 찾을 수 있을 거예요. `int`는 정말로 바꿀 수 없는 값을 받으면 이런 식으로 오류가 생겼다고 알려주면서 프로그램이 종료돼요. 이런 에러는 프로그램에 아주 치명적이에요. 그래서 에러를 잡아주는 코드를 넣어서 에러가 생기지 않게 잡아줄 거예요.

<span id="try_except" />

## try...except

```python
try:
    age = int(input("age? :"))
    print(type(age))
except ValueError:
    print("숫자를 입력하세요")
```

에러를 잡아주기 위해서 `try except`를 추가했어요. `try` 안에는 에러가 생길 수도 있는 코드를 넣어요. 지금은 `int()` 함수가 에러를 만들 수 있으니까 `try` 안에 넣어줬어요. 그리고 이 코드에서 에러가 생기면 `except` 안에 있는 코드가 실행돼요.

`except` 다음에 오는 `ValueError` 코드는 지금 상황에서 넣지 않아도 잘 작동합니다. `except ValueError`는 에러의 종류가 `ValueError`인 경우에 실행하세요 라는 의미에요. 만일 `ValueError`를 넣지 않으면 모든 에러 상황에 실행돼요.

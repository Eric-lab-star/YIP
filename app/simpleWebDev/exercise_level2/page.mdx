<MissionCard number={1} title="투표 집계기" time="12분">

세 후보 중 하나에 투표하면, 각 후보의 표가 쌓이는 앱이다냥.

- `session_state` 에 후보별 표 수를 딕셔너리로 저장한다냥.
- 버튼 세 개(후보 A/B/C)를 만들고, 누르면 해당 후보의 표를 1 늘린다냥.
- 현재 집계를 아래에 보여준다냥.

```python
import streamlit as st

if "votes" not in st.session_state:
    st.session_state["votes"] = {"A": 0, "B": 0, "C": 0}

st.title("최애 간식 투표다냥")

col1, col2, col3 = st.columns(3)
with col1:
    if st.button("참치캔"):
        st.session_state["votes"]["A"] += 1
with col2:
    if st.button("닭가슴살"):
        st.session_state["votes"]["B"] += 1
with col3:
    if st.button("츄르"):
        st.session_state["votes"]["C"] += 1

st.write("참치캔: " + str(st.session_state["votes"]["A"]) + "표")
st.write("닭가슴살: " + str(st.session_state["votes"]["B"]) + "표")
st.write("츄르: " + str(st.session_state["votes"]["C"]) + "표")
```

<Callout type="tip">
  카운터가 세 개로 늘어난 것뿐이라냥. 딕셔너리 하나에 세 후보를 담으니
  깔끔하다냥!
</Callout>

</MissionCard>

<MissionCard number={2} title="할 일 목록 (심화)" time="15분">

폼으로 할 일을 받아서 리스트에 쌓는 앱이다냥. 방명록과 거의 같지만,
직접 처음부터 설계해보자냥.

<Callout type="important">
  **스스로 설계하기 — 순서만 알려준다냥**

  1. `session_state` 에 빈 리스트 `todos` 를 초기화한다냥.
  2. `st.form` 안에 `st.text_input` 으로 할 일을 받고, 제출 버튼을 둔다냥.
  3. 제출되고 내용이 비어 있지 않으면 리스트에 `append` 한다냥.
  4. `for` 로 할 일들을 화면에 보여준다냥.
  5. (도전) `st.metric` 으로 "남은 할 일 개수" 를 보여준다냥.

  막히면 13강(방명록)과 14강(폼)을 다시 보면 된다냥!
</Callout>

</MissionCard>

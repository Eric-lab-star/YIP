## 1단계 · 한 가지 규칙만 기억하자냥

<Callout type="important">
  **Streamlit 의 황금 규칙**

  화면에서 **무언가 하나라도 건드리면**(글자를 치든, 버튼을 누르든,
  슬라이더를 밀든), Streamlit 은 **`app.py` 를 처음부터 끝까지 통째로 다시
  실행한다냥.**

  일부만 바꾸는 게 아니라 **전부 다시** 실행하는 거다냥. 이게 핵심이라냥!
</Callout>

<NyangSpeech>
  놀랐냥? 우리가 앞에서 "입력창에 글자를 칠 때마다 아래 화면이 자동으로
  바뀐다" 고 했지? 그게 바로 이 규칙 때문이다냥. 글자를 한 자 칠 때마다
  코드 전체가 다시 돌아서 화면을 새로 그리는 거다냥!
</NyangSpeech>

## 3단계 · 버튼은 "눌린 그 순간만" True 다냥

<NyangSpeech>
  이제 버튼을 보자냥. `st.button` 은 **눌린 바로 그 순간의 재실행에서만
  True** 가 되고, 그 다음 재실행부터는 다시 False 로 돌아간다냥. 코드로
  확인해보자냥.
</NyangSpeech>

```python
import streamlit as st

st.title("버튼 실험이다냥")

if st.button("눌러보기"):
    st.write("지금 막 눌렀다냥!")   # 누른 그 순간에만 보인다냥

st.write("이 줄은 항상 보인다냥")
```

<Callout type="note">
  **직접 해보면 이렇다냥**

  - 버튼을 누르면 → "지금 막 눌렀다냥!" 이 뜬다냥.
  - 그런데 슬라이더를 밀거나 다른 걸 건드리면 → 그 메시지가 **사라진다냥.**

  버튼이 False 로 돌아갔기 때문이라냥. 버튼은 "방금 눌렸냐?" 를 알려주는
  거지, "예전에 눌렀었냐?" 를 기억하는 게 아니라냥!
</Callout>

## 4단계 · "값이 사라져요" 의 정체

<NyangSpeech>
  많은 사람이 여기서 처음 좌절한다냥. 예를 들어 버튼을 누를 때마다 숫자를
  1씩 늘리고 싶어서 이렇게 짠다냥.
</NyangSpeech>

```python
import streamlit as st

count = 0                    # 매번 0으로 다시 시작한다냥!

if st.button("증가"):
    count = count + 1

st.write("횟수: " + str(count))
```

<Callout type="important">
  **왜 항상 1에서 멈추냥?**

  버튼을 눌러도 계속 "횟수: 1" 만 나온다냥. 왜냐면 **재실행 때마다
  `count = 0` 부터 다시 시작**하기 때문이라냥. 눌러서 1이 됐다가, 다음
  재실행에서 또 0으로 리셋되는 거다냥.

  이건 버그가 아니라 **재실행 규칙 그대로**라냥. 값을 기억하게 하려면
  특별한 상자가 필요하다냥 — 그게 바로 다음 3부에서 배울
  **`st.session_state`** 라냥!
</Callout>

---

## 5단계 · 그래서 무엇을 조심하냥

<Callout type="tip">
  **재실행 시대의 마음가짐**

  - 일반 변수(`count = 0`)는 **재실행마다 초기화**된다냥. 기억 못 한다냥.
  - `st.button` 은 **눌린 순간만** True 라냥. 상태를 저장하는 게 아니라냥.
  - 값을 **계속 기억**하게 하려면 `st.session_state` 가 필요하다냥(3부).

  지금은 "아, 화면을 건드리면 전부 다시 도는구나" 만 확실히 알면 된다냥!
</Callout>
